# SPEC-00: 架构总则

> 定义 NcatBot v5.0 的分层架构模型、层间依赖规则和目录结构规范。

## 1. 四层架构模型

```
┌──────────────────────────────────────────────────────────────────┐
│  Layer 4: Plugin System（插件层）                                │
│  BasePlugin · PluginLoader · NcatBotPlugin · 用户插件            │
│  依赖：IBotAPI · EventBus · ServiceManager · RegistryEngine     │
├──────────────────────────────────────────────────────────────────┤
│  Layer 3: Core（核心层）                                         │
│  ┌──────────────┐ ┌───────────────┐ ┌──────────────────────┐    │
│  │  EventBus    │ │RegistryEngine │ │  ServiceManager      │    │
│  │  Dispatcher  │ │ CommandRunner │ │  (纯内部服务)         │    │
│  │  NcatEvent   │ │ FilterRunner  │ │  RBAC · Config ·     │    │
│  │              │ │ Executor      │ │  Data · Watcher ·    │    │
│  │              │ │               │ │  TimeTask            │    │
│  └──────────────┘ └───────────────┘ └──────────────────────┘    │
├──────────────────────────────────────────────────────────────────┤
│  Layer 2: Abstract API（抽象 API 层）                            │
│  IBotAPI 接口 · 标准 Event 模型 · API 错误类型 · 响应模型        │
├──────────────────────────────────────────────────────────────────┤
│  Layer 1: Adapter（平台适配层）                                  │
│  ┌────────────────────────────────────────────────────────┐     │
│  │  NapCatAdapter (OneBot v11)                             │     │
│  │  ├─ Environment  : NapCat 安装/部署/启动                │     │
│  │  ├─ Connection   : WebSocket 连接管理                   │     │
│  │  ├─ Protocol     : echo/Future 请求-响应匹配            │     │
│  │  ├─ EventConvert : 原始 JSON → 标准 Event               │     │
│  │  ├─ APIImpl      : IBotAPI 的 OneBot v11 实现           │     │
│  │  ├─ PreUpload    : 文件预上传（平台特定）               │     │
│  │  └─ Auth         : 登录认证                             │     │
│  └────────────────────────────────────────────────────────┘     │
└──────────────────────────────────────────────────────────────────┘
```

### 1.1 各层定位

| 层 | 定位 | 关键词 |
|----|------|--------|
| Layer 4 Plugin | 用户扩展点，通过框架提供的接口开发功能 | 扩展、命令、过滤器 |
| Layer 3 Core | 框架核心引擎，平台无关的事件处理和服务管理 | EventBus、注册引擎、内部服务 |
| Layer 2 Abstract API | 框架与外部的交互契约定义，不含任何实现 | 接口、模型、契约 |
| Layer 1 Adapter | 特定平台的完整适配实现 | 连接、协议、转换 |

### 1.2 "内部"与"外部"的判定标准

| 判定维度 | 内部（Core/Service） | 外部（Adapter） |
|---------|---------------------|----------------|
| 是否依赖特定通信协议 | 否 | 是（WebSocket/HTTP/...） |
| 是否依赖特定平台数据格式 | 否 | 是（OneBot JSON/...） |
| 是否涉及网络 I/O | 否 | 是 |
| 是否需要平台环境 | 否 | 是（NapCat 安装/启动） |
| 移除后框架是否可编译运行 | 是（核心功能不受影响） | 否（无法通信） |

## 2. 层间依赖规则

### 2.1 依赖方向

```
Layer 4 (Plugin) ──依赖──→ Layer 3 (Core) ──依赖──→ Layer 2 (Abstract API)
                                                            ↑
Layer 1 (Adapter) ──────────────实现────────────────────────┘
```

### 2.2 具体规则

| 规则 | 说明 | 示例 |
|------|------|------|
| **D-1** 上层可依赖下层 | Core 可 import Abstract API 的类型 | `from core.api import IBotAPI` ✅ |
| **D-2** 下层不可依赖上层 | Adapter 不可 import Core 的类 | `from core.client import EventBus` ❌ |
| **D-3** 同层通过接口交互 | Core 引用 IBotAPI，不引用具体实现 | `api: IBotAPI` ✅ / `api: NapCatBotAPI` ❌ |
| **D-4** Adapter 仅依赖 Layer 2 | Adapter 实现 IBotAPI，使用标准 Event | `class NapCatBotAPI(IBotAPI)` ✅ |
| **D-5** Service 不持有 Adapter 引用 | Service 通过 EventBus 或 IBotAPI 间接通信 | Service 内无 WebSocket 引用 ✅ |
| **D-6** Plugin 不直接触碰 Adapter | Plugin 通过 IBotAPI + EventBus + ServiceManager | `self.api.send_group_msg()` ✅ |

### 2.3 依赖例外

| 例外 | 说明 |
|------|------|
| **E-1** Layer 1 可依赖 Layer 2 的标准 Event 模型 | Adapter 需要构造标准 Event 对象 |
| **E-2** BotClient 作为顶层编排者可同时引用 Layer 1 和 Layer 3 | BotClient 负责组装 Adapter 和 Core |
| **E-3** TYPE_CHECKING 块中允许跨层导入用于类型提示 | 仅用于 IDE 支持，不影响运行时 |

### 3.1 目录命名规范

| 规则 | 说明 |
|------|------|
| 使用小写 + 下划线 | `message_segments.py`，不用驼峰 |
| 包目录必须有 `__init__.py` | 控制公共 API 导出 |
| 一个文件一个核心类 | 避免单文件过大，便于 import |
| 测试镜像源码结构 | `test/core/service/test_rbac.py` |

### 3.2 导出规范

每个包的 `__init__.py` 必须通过 `__all__` 显式声明公共 API：

```python
# ncatbot/core/api/__init__.py
from .interface import IBotAPI
from .client import IAPIClient, APIResponse, CallbackAPIClient
from .errors import APIError

__all__ = ["IBotAPI", "IAPIClient", "APIResponse", "CallbackAPIClient", "APIError"]
```

## 4. 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 抽象基类 | `Base` 或 `I` 前缀 | `BaseAdapter`、`IBotAPI` |
| 具体实现 | 平台名 + 功能名 | `NapCatAdapter`、`NapCatBotAPI` |
| 服务类 | 功能名 + `Service` | `RBACService`、`PluginConfigService` |
| 引擎/核心组件 | 功能名 + `Engine`/`Runner`/`Executor` | `RegistryEngine`、`CommandRunner` |
| 事件类 | 动作/类型 + `Event` | `GroupMessageEvent`、`PokeNotifyEvent` |
| 枚举 | 域名 + `Type`/`SubType` | `PostType`、`MessageType` |
| 接口/协议 | `I` 前缀 或 Protocol | `IBotAPI`、`IAPIClient` |
