# SPEC-09: 模块依赖分析与接口提取

## 1. 概述

本文对 `ncatbot_legacy/` 中所有核心模块进行引用与反向引用分析，列出每个模块**真正需要的能力**，设计 Protocol 接口与依赖注入方案。

### 4.1 BotClient

**删除 Legacy 后持有的属性**:

| 属性 | 类型 | 创建方式 |
|------|------|----------|

## 6. 接口协议设计

### 6.1 IPluginRegistry

**解决**: CommandRunner 对 BotClient 的依赖（插件查询）

```python
from typing import Protocol, Type, TypeVar, List, runtime_checkable, TYPE_CHECKING

if TYPE_CHECKING:
    from ncatbot.plugin_system.base_plugin import BasePlugin

T = TypeVar("T")

@runtime_checkable
class IPluginRegistry(Protocol):
    """插件查询协议"""

    def get_plugin(self, plugin_type: Type[T]) -> T: ...
    def get_plugin_class_by_name(self, plugin_name: str) -> Type["BasePlugin"]: ...
    def get_registered_plugins(self) -> List["BasePlugin"]: ...
```

**实现者**: `BotClient`（已天然实现）

**消费者**: `CommandRunner.run()` — `bot_client` 参数 → `plugin_registry: IPluginRegistry`

---

### 6.2 IHandlerDispatcher

**解决**: NcatBotPlugin 对 `bot_client.handler_dispatcher` 的依赖

```python
from typing import Protocol, Callable, Any

class IHandlerDispatcher(Protocol):
    """Handler 注册/清理协议"""

    def register_handler(
        self,
        event_type: str,
        func: Callable,
        priority: int = 0,
        plugin_name: str = "",
        **metadata: Any,
    ) -> Any: ...

    def revoke_plugin(self, plugin_name: str) -> int: ...
```

**实现者**: `HandlerDispatcher`（已天然满足）

**消费者**:
- `NcatBotPlugin.__onload__()` — `flush_pending(dispatcher, name)`
- `NcatBotPlugin.__unload__()` — `dispatcher.revoke_plugin(name)`

---

**现有定义**: `core/api/interface.py` 中的 `IBotAPI`

**消费者**: `PluginLoader._load_plugin_by_class()` — `plugin.api = services.api`

## 7. 依赖注入方案

### 7.1 ServiceManager 改造

新增窄接口属性，替代 `_bot_client` 万能引用：

```python
class ServiceManager:
    def __init__(self, event_bus: "EventBus"):
        self._event_bus = event_bus
        self._api: Optional["IBotAPI"] = None
        self._handler_dispatcher: Optional[IHandlerDispatcher] = None
        self._plugin_registry: Optional[IPluginRegistry] = None

    @property
    def api(self) -> "IBotAPI":
        return self._api

    @property
    def handler_dispatcher(self) -> IHandlerDispatcher:
        return self._handler_dispatcher

    @property
    def plugin_registry(self) -> Optional[IPluginRegistry]:
        return self._plugin_registry

    def set_api(self, api: "IBotAPI") -> None:
        self._api = api

    def set_handler_dispatcher(self, d: IHandlerDispatcher) -> None:
        self._handler_dispatcher = d

    def set_plugin_registry(self, r: IPluginRegistry) -> None:
        self._plugin_registry = r
```

**删除**: `set_bot_client()` / `_bot_client` / `bot_client` 属性、`event_bus` 属性中的 `_bot_client` fallback。

### 7.2 BotClient 注入时序

```python
class BotClient(LifecycleManager):
    def __init__(self, adapter=None):
        self.event_bus = EventBus()
        self.handler_dispatcher = HandlerDispatcher(self.event_bus)
        self.services = ServiceManager(event_bus=self.event_bus)

        # 注入窄接口（单向，无循环引用）
        self.services.set_handler_dispatcher(self.handler_dispatcher)
        ...

    async def _setup_api(self):
        self.api = self.adapter.get_api()
        self.dispatcher = EventDispatcher(self.event_bus, self.api, services=self.services)
        self.adapter.set_event_callback(self.dispatcher.dispatch)

        # 延迟注入（需要 api/plugin_loader ready）
        self.services.set_api(self.api)
        self.services.set_plugin_registry(self)
```

### 7.3 消费方改造清单

| 文件 | 旧调用 | 新调用 |
|------|--------|--------|
| `loader/core.py` | `services.bot_client.api` | `services.api` |
| `ncatbot_plugin.py` | `services.bot_client.handler_dispatcher` | `services.handler_dispatcher` |
| `command_runner.py` | `bot_client` 参数 | `plugin_registry: IPluginRegistry` 参数 |
| `base.py` | `service_manager._bot_client.event_bus` fallback | 删除，仅用 `service_manager._event_bus` |
| `manager.py` | `_bot_client.event_bus` fallback | 删除，仅用 `_event_bus` |

## 8. 改造后的依赖图

```
改造前 (循环):
  BotClient ←──→ ServiceManager ←──→ PluginLoader

改造后 (单向):
  BotClient ──→ ServiceManager ──→ { IHandlerDispatcher, IPluginRegistry, IBotAPI, EventBus }
  BotClient ──→ HandlerDispatcher ──→ EventBus
  BotClient ──→ EventDispatcher ──→ { EventBus, IBotAPI }
  BotClient ──→ Adapter
```
