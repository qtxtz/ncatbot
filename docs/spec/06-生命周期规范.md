# SPEC-06: 生命周期规范

> 定义 BotClient 的初始化、启动、运行、关闭的完整生命周期流程，以及事件和 API 调用的数据流。

## 1. 初始化阶段

### 1.1 BotClient.__init__()

```
BotClient.__init__()
│
├── 1. 创建 EventBus
│       event_bus = EventBus()
│
├── 2. 创建 RegistryEngine
│       registry = RegistryEngine()
│
├── 3. RegistryEngine 订阅 EventBus
│       event_bus.subscribe("re:ncatbot\\..*message.*", registry.handle_message_event)
│       event_bus.subscribe("re:ncatbot\\.notice_event|...", registry.handle_legacy_event)
│
├── 4. 创建 ServiceManager（注入 EventBus）
│       services = ServiceManager(event_bus)
│
├── 5. 注册内置 Service
│       services.register(RBACService)
│       services.register(PluginConfigService)
│       services.register(PluginDataService)
│       services.register(FileWatcherService)
│       services.register(TimeTaskService)
│
└── 6. 创建/选择 Adapter
        adapter = NapCatAdapter()  # 默认，或通过参数选择
```

### 1.2 与现有 __init__() 的对比

| 现有 | 新版 | 变更原因 |
|------|------|---------|
| 注册 MessageRouter | **移除** | → Adapter |
| 注册 PreUploadService | **移除** | → Adapter |
| 注册 UnifiedRegistryService | **移除** | → RegistryEngine |
| 无 RegistryEngine | 创建并订阅 EventBus | 替代 SystemManager 桥接 |
| ServiceManager() 无参 | ServiceManager(event_bus) | 注入 EventBus |
| 无 Adapter 概念 | 创建并持有 Adapter | 新增 |

## 2. 启动阶段

### 2.1 BotClient.run() / run_backend_async()

```
BotClient.run(**kwargs)
│
├── 准备阶段
│   ├── 解析启动参数（bot_uin, ws_uri, ...）
│   ├── 验证配置
│   └── 创建 PluginLoader
│
└── _core_execution()
    │
    ├── Step 1: event_bus.bind_loop()
    │       绑定当前事件循环，支持跨线程发布
    │
    ├── Step 2: adapter.setup()                    ← 新增
    │       准备平台环境（安装/启动 NapCat）
    │       替代现有的 launch_napcat_service()
    │
    ├── Step 3: adapter.connect()                  ← 新增
    │       建立 WebSocket 连接
    │       替代现有 MessageRouter.on_load() 中的连接逻辑
    │
    ├── Step 4: services.load_all()
    │       加载所有内部服务（RBAC/Config/Data/...）
    │       不再包含 MessageRouter/PreUpload/UnifiedRegistry
    │
    ├── Step 5: api = adapter.get_api()            ← 新增
    │       获取 IBotAPI 实现
    │       替代现有 BotAPI(router.send, service_manager)
    │
    ├── Step 6: dispatcher = EventDispatcher(event_bus, api)
    │       创建事件分发器
    │
    ├── Step 7: adapter.set_event_callback(dispatcher.dispatch)  ← 新增
    │       将 Adapter 的事件输出连接到 Core 的事件分发
    │       替代现有 router.set_event_dispatcher(dispatcher)
    │
    ├── Step 8: registry.set_plugin_resolver(plugin_loader.get)  ← 新增
    │       注入插件实例解析器
    │
    ├── Step 9: plugin_loader.load_all()
    │       加载外部插件和内置插件
    │
    ├── Step 10: startup_event.set()  （仅 run_backend_async）
    │       通知启动完成，返回 api
    │
    └── Step 11: adapter.listen()                  ← 新增
            开始监听（阻塞）
            替代现有 router.websocket.listen()
```

### 2.2 与现有 _core_execution() 的对比

| 步骤 | 现有实现 | 新版实现 |
|------|---------|---------|
| 平台环境 | `launch_napcat_service()` | `adapter.setup()` |
| 连接建立 | `MessageRouter.on_load()` 内部 | `adapter.connect()` |
| 服务加载 | `services.load_all()` 含 8 个服务 | `services.load_all()` 含 5 个服务 |
| API 构建 | `BotAPI(router.send, services)` | `adapter.get_api()` |
| 事件回调 | `router.set_event_dispatcher(d)` | `adapter.set_event_callback(d.dispatch)` |
| 监听 | `router.websocket.listen()` | `adapter.listen()` |

## 3. 运行阶段

### 3.1 事件流

```
外部平台 (NapCat/OneBot v11)
    │
    │  WebSocket 消息
    ▼
Adapter._on_raw_message(raw_data: dict)
    │
    ├── [API 响应] echo 字段存在
    │   └── protocol.resolve(echo, data)
    │       └── asyncio.Future.set_result(data)
    │           └── 等待中的 API 调用返回结果
    │
    └── [事件] echo 字段不存在
        ├── event = adapter.convert_event(raw_data)
        │   ├── EventConverter 查找 Event 类
        │   ├── pydantic 构造 Event 对象
        │   └── event.bind_api(adapter.get_api())
        │
        └── await event_callback(event)
            │
            ▼
        EventDispatcher.dispatch(event)
            ├── ncatbot_event = NcatBotEvent(type, event)
            └── event_bus.publish(ncatbot_event)
                │
                ├── [精确匹配] "ncatbot.group_message_event"
                │   └── 用户注册的 handler
                │
                ├── [正则匹配] "re:ncatbot\\..*message.*"
                │   └── RegistryEngine.handle_message_event(event)
                │       ├── CommandRunner.run(event)
                │       │   └── 匹配命令 → 执行处理函数
                │       └── run_pure_filters(event)
                │           └── 遍历过滤器 → 执行过滤器函数
                │
                └── [正则匹配] "re:ncatbot\\.notice_event|..."
                    └── RegistryEngine.handle_legacy_event(event)
                        └── 分发到 notice/request/meta 处理器
```

### 3.2 API 调用流

```
plugin / event handler
    │
    │  event.api.send_group_msg(group_id, message)
    │  或 api.send_group_msg(group_id, message)
    ▼
IBotAPI.send_group_msg()                    # 抽象接口
    │
    ▼
NapCatBotAPI.send_group_msg()              # Adapter 具体实现
    │
    ├── preupload_message(message)          # 预上传文件（可选）
    │
    ├── NapCatProtocol.send(                # 协议层
    │       action="send_group_msg",
    │       params={"group_id": ..., "message": ...},
    │   )
    │   ├── 生成 UUID echo
    │   ├── 创建 asyncio.Future
    │   ├── NapCatConnection.send(data)     # WebSocket 发送
    │   └── await future (等待响应)
    │
    ▼
NapCat 服务 (外部平台)
    │
    │  WebSocket 响应 (含 echo)
    ▼
Adapter._on_raw_message(response)
    └── protocol.resolve(echo, data)
        └── Future.set_result(data)
            └── NapCatProtocol.send() 返回结果
```

## 4. 关闭阶段

### 4.1 BotClient.shutdown()

```
BotClient.shutdown()
│
├── Step 1: plugin_loader.unload_all()
│       卸载所有插件
│       RegistryEngine.handle_plugin_unload() 清理注册项
│
├── Step 2: services.close_all()
│       关闭所有内部服务
│       按注册顺序的逆序关闭
│
├── Step 3: adapter.disconnect()               ← 新增
│       断开平台连接
│       ├── protocol.cancel_all()  取消所有挂起的请求
│       └── connection.disconnect()  关闭 WebSocket
│
└── Step 4: registry.clear()                   ← 新增
        清理 RegistryEngine 状态
```

### 4.2 与现有 _cleanup() 的对比

| 步骤 | 现有 | 新版 |
|------|------|------|
| 卸载插件 | `plugin_loader.unload_all()` | 相同 |
| 关闭服务 | `services.close_all()`（含 WebSocket 关闭） | `services.close_all()` + `adapter.disconnect()` |
| 清理 Registry | 无 | `registry.clear()` |

## 5. 错误处理

### 5.1 启动阶段错误

```python
async def _core_execution(self):
    try:
        await adapter.setup()
    except AdapterSetupError as e:
        LOG.error(f"平台环境准备失败: {e}")
        raise

    try:
        await adapter.connect()
    except AdapterConnectionError as e:
        LOG.error(f"连接失败: {e}")
        raise

    try:
        await services.load_all()
    except Exception as e:
        LOG.error(f"服务加载失败: {e}")
        await adapter.disconnect()
        raise

    # ... 后续步骤
```

### 5.2 运行阶段错误

- **连接中断**：`adapter.listen()` 抛出 `AdapterConnectionError`，触发 `_cleanup()`
- **取消信号**：`asyncio.CancelledError` 正常退出
- **未捕获异常**：记录日志后触发 `_cleanup()`

## 6. 启动模式

| 模式 | 方法 | 说明 |
|------|------|------|
| 同步阻塞 | `bot.run()` | CLI 使用，`asyncio.run()` |
| 异步非阻塞 | `bot.run_backend_async()` | 返回 `IBotAPI`，Bot 后台运行 |
| 同步非阻塞 | `bot.run_backend()` | 新线程中启动事件循环 |

三种模式均走同一个 `_core_execution()` 流程。

## 7. 生命周期合规检查清单

- [ ] Adapter 生命周期：setup → connect → listen → disconnect
- [ ] Service 生命周期：register → load_all → close_all
- [ ] RegistryEngine 生命周期：subscribe → initialize → clear
- [ ] Plugin 生命周期：load → unload（在 Adapter 和 Service 之后加载，之前卸载）
- [ ] 启动失败时正确清理已分配的资源
- [ ] `_cleanup()` 幂等（可安全多次调用）
- [ ] 不再调用 `launch_napcat_service()`（由 `adapter.setup()` 替代）
- [ ] 不再引用 `router.websocket`（由 `adapter.listen()` 替代）
