# 服务层（Service Layer）

服务层是 NcatBot 的基础设施层，为上层（客户端、插件系统、API）提供可插拔的底层能力支撑。所有外部依赖（WebSocket 通信、文件上传、权限管理、定时任务等）均以服务形式封装，通过统一的 `ServiceManager` 管理生命周期。

## 目录结构

```
ncatbot/core/service/
├── __init__.py
├── base.py                    # BaseService 抽象基类
├── manager.py                 # ServiceManager 服务管理器
└── builtin/                   # 内置服务
    ├── __init__.py
    ├── message_router.py      # 消息路由服务
    ├── preupload/             # 文件预上传服务
    ├── plugin_config/         # 插件配置服务
    ├── plugin_data/           # 插件数据持久化服务
    ├── rbac/                  # 权限控制服务
    ├── file_watcher/          # 文件监控服务
    ├── time_task/             # 定时任务服务
    └── unified_registry/      # 统一注册服务
```

## 1. BaseService 抽象基类

所有服务必须继承 `BaseService`，并实现生命周期钩子方法。

### 类定义

```python
class BaseService(ABC):
    name: str = None          # 子类必须定义，作为服务唯一标识
    description: str = "未提供描述"

    def __init__(self, **config: Any):
        if not self.name:
            raise ValueError(f"{self.__class__.__name__} 必须定义 name 属性")
        self.config = config
        self._loaded = False
        self.service_manager: Optional["ServiceManager"] = None
```

### 生命周期钩子

| 方法 | 说明 |
|------|------|
| `on_load()` | 异步加载，资源分配、连接建立等 |
| `on_close()` | 异步关闭，资源释放、连接关闭等 |

### 内部方法与属性

| 成员 | 类型 | 说明 |
|------|------|------|
| `_load()` | async | 内部加载方法，防重复加载 |
| `_close()` | async | 内部关闭方法，防重复关闭 |
| `is_loaded` | property | 服务是否已加载 |
| `event_bus` | property | 通过 `service_manager.bot_client` 获取事件总线 |

### 生命周期流程

```
__init__(**config)     →  实例化（轻量级，仅保存配置）
    │
    ▼
_load() → on_load()   →  异步加载（资源分配、连接建立）
    │
    ▼
  [运行中]             →  服务可用，处理业务逻辑
    │
    ▼
_close() → on_close() →  异步关闭（资源释放、连接关闭）
```

## 2. ServiceManager 服务管理器

`ServiceManager` 负责所有服务的注册、加载、卸载和访问。它是服务层的统一入口。

### 核心属性

```python
class ServiceManager:
    _debug_mode: bool = False
    _test_mode: bool = False

    def __init__(self):
        self._services: Dict[str, BaseService] = {}         # 已加载的服务实例
        self._service_classes: Dict[str, Type[BaseService]] = {}   # 已注册的服务类
        self._service_configs: Dict[str, Dict[str, Any]] = {}     # 服务配置
        self._bot_client: Optional["BotClient"] = None
```

### 管理方法

| 方法 | 签名 | 说明 |
|------|------|------|
| `register` | `(service_class, **config)` | 注册服务类（不立即实例化） |
| `load` | `async (service_name) → BaseService` | 加载指定服务（创建实例 + 调用 `on_load`） |
| `unload` | `async (service_name)` | 卸载服务（调用 `on_close` + 移除实例） |
| `load_all` | `async ()` | 加载所有已注册但未加载的服务 |
| `close_all` | `async ()` | 关闭所有已加载的服务 |
| `get` | `(service_name) → Optional[BaseService]` | 获取服务实例 |
| `has` | `(service_name) → bool` | 检查服务是否已加载 |
| `list_services` | `() → List[str]` | 列出所有已加载服务名称 |

### 配置与环境方法

| 方法 | 说明 |
|------|------|
| `set_debug_mode(enable)` | 设置调试模式 |
| `set_test_mode(enable)` | 设置测试模式 |
| `set_bot_client(bot_client)` | 注入 BotClient 引用 |

### IDE 友好的属性访问

ServiceManager 为每个内置服务提供了类型提示属性，可直接通过属性名访问，IDE 能自动补全：

```python
manager = ServiceManager()

# 类型安全访问（IDE 支持自动补全）
router = manager.message_router       # 类型: MessageRouter
preupload = manager.preupload         # 类型: PreUploadService
config = manager.plugin_config        # 类型: PluginConfigService
rbac = manager.rbac                   # 类型: RBACService
registry = manager.unified_registry   # 类型: UnifiedRegistryService
watcher = manager.file_watcher        # 类型: FileWatcherService
data = manager.plugin_data            # 类型: PluginDataService
```

### 注册与加载流程

```
register(ServiceClass, **config)
    │   保存 class 和 config
    ▼
load(service_name)
    │   1. 检查是否已加载（去重）
    │   2. 实例化 service_class(**config)
    │   3. 注入 service_manager 到服务实例
    │   4. 调用 service._load() → on_load()
    │   5. 存入 _services 字典
    ▼
[服务可用]
    │
    ▼
unload(service_name)
    │   1. 调用 service._close() → on_close()
    │   2. 从 _services 中移除
    ▼
[服务已卸载]
```

### 3.1 MessageRouter — 消息路由服务

**文件**: `ncatbot/core/service/builtin/message_router.py`

消息路由服务负责 WebSocket 消息的发送和接收，是 API 调用的底层通道。它采用 **Echo-based 请求-响应匹配** 模式，将异步 API 调用映射为 `asyncio.Future`。

#### 核心机制

```
API 调用 → send(action, params)
    │
    │  1. 生成唯一 UUID 作为 echo
    │  2. 创建 asyncio.Future，存入 _pending_futures[echo]
    │  3. 通过 WebSocket 发送 {action, params, echo}
    │  4. await Future（带超时）
    │
    ▼ WebSocket 收到消息
_on_message(data)
    │
    ├── 有 echo 字段 → API 响应 → 设置对应 Future 结果
    │
    └── 无 echo 字段 → 事件推送 → 转发给 EventDispatcher
```

#### 关键方法

| 方法 | 签名 | 说明 |
|------|------|------|
| `send` | `async (action, params, timeout=30.0) → dict` | 发送 API 请求并等待响应 |
| `_on_message` | `(data)` | 处理 WebSocket 消息，区分响应和事件 |
| `start_listening` | `()` | 开始监听 WebSocket |
| `set_event_dispatcher` | `(dispatcher)` | 设置事件消息的处理回调 |

#### 线程安全设计

```python
_pending_futures: Dict[str, asyncio.Future]  # echo → 待处理的 Future
_futures_lock: threading.Lock                 # 保护并发访问
```

- 使用 `threading.Lock` 保护 `_pending_futures` 字典
- 通过 `call_soon_threadsafe` 在事件循环中安全地设置 Future 结果
- 支持跨线程的请求-响应匹配

#### 配置参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `uri` | `ncatbot_config.get_uri_with_token()` | WebSocket 连接 URI |
| `event_dispatcher` | - | 事件消息的异步回调 |

### 3.2 PreUploadService — 预上传服务

**文件**: `ncatbot/core/service/builtin/preupload/`

预上传服务提供统一的文件和消息预上传能力，支持多种输入格式的自动检测和处理。

#### 支持的文件类型

- `image`（图片）、`video`（视频）、`record`（语音）、`file`（通用文件）

#### 输入格式自动检测

| 格式 | 示例 | 处理方式 |
|------|------|----------|
| 本地文件 | `/path/to/file.png`、`file://path` | 分块上传 |
| Base64 数据 | `base64://iVBOR...` | 解码后上传 |
| 远程 URL | `https://example.com/img.png` | 直接透传（不上传） |

#### 关键方法

| 方法 | 签名 | 说明 |
|------|------|------|
| `upload_file` | `(file_path) → UploadResult` | 上传本地文件（分块） |
| `upload_bytes` | `(data, filename) → UploadResult` | 上传二进制数据 |
| `process_message` | `(data) → ProcessResult` | 预上传消息中所有文件 |
| `preupload_file` | `(file_value, file_type) → PreUploadResult` | 预上传单个文件 |
| `preupload_file_if_needed` | `(file_value) → str` | 按需预上传，返回路径 |

#### 配置参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `chunk_size` | `500 * 1024`（500 KB） | 分块上传大小 |
| `file_retention` | `600 * 1000`（600 秒） | 文件保留时间（毫秒） |

#### 架构组件

- **StreamUploadClient**：处理分块上传协议
- **MessagePreUploadProcessor**：遍历消息结构，识别文件字段并预上传

#### 依赖关系

依赖 `message_router` 服务进行 WebSocket 通信。

### 3.3 PluginConfigService — 插件配置服务

**文件**: `ncatbot/core/service/builtin/plugin_config/`

为所有插件提供统一的配置注册、读写和持久化管理。每个插件的配置以独立 YAML 文件存储。

#### ConfigItem 数据结构

```python
class ConfigItem:
    name: str                          # 配置项名称
    default_value: Any                 # 默认值
    description: str                   # 描述
    value_type: type                   # 值类型
    metadata: Dict[str, Any]           # 自定义元数据
    plugin_name: str                   # 所属插件名
    on_change: Optional[Callable]      # 值变更时的回调
```

#### 关键方法

| 方法 | 签名 | 说明 |
|------|------|------|
| `register_config` | `(plugin, name, default, desc, type, ...) → ConfigItem` | 注册配置项（可附带变更回调） |
| `get` | `(plugin, name, default=None) → Any` | 获取配置值 |
| `set` | `(plugin, name, value) → (old, new)` | 设置配置值（延迟保存） |
| `set_atomic` | `(plugin, name, value) → (old, new)` | 设置并立即持久化 |
| `set_atomic_async` | `async (plugin, name, value) → (old, new)` | 异步原子保存 |
| `get_plugin_config` | `(plugin) → Dict` | 获取插件全部配置（副本） |
| `get_plugin_config_wrapper` | `(plugin) → PluginConfig` | 获取只读配置包装器 |
| `set_plugin_config` | `(plugin, config)` | 批量设置（延迟保存） |
| `set_plugin_config_atomic` | `(plugin, config)` | 批量原子保存 |
| `unregister_plugin_configs` | `(plugin)` | 清理失效配置（插件卸载时） |
| `reload_plugin_config` | `(plugin) → Dict` | 从文件重新加载 |

#### 持久化策略

| 模式 | 方法 | 时机 | 说明 |
|------|------|------|------|
| **延迟保存** | `set()` | 服务关闭时统一保存 | 性能优先，标记脏数据 |
| **原子保存** | `set_atomic()` | 调用时立即写入文件 | 安全优先，防数据丢失 |
| **迁移** | 自动 | 加载时 | 从旧版 YAML 配置迁移 |

#### 使用示例

```python
class MyPlugin(BasePlugin):
    async def on_load(self):
        # 注册配置项
        self.config_service.register_config(
            self, "max_retry", 3,
            description="最大重试次数",
            value_type=int,
            on_change=self._on_retry_change
        )

    def _on_retry_change(self, old, new):
        self.logger.info(f"重试次数从 {old} 变更为 {new}")
```

### 3.4 PluginDataService — 插件数据服务

**文件**: `ncatbot/core/service/builtin/plugin_data/`

为插件提供 JSON 格式的数据持久化能力，支持内存缓存。

#### 关键方法

| 方法 | 签名 | 说明 |
|------|------|------|
| `load_plugin_data` | `(plugin_name) → Dict` | 从 JSON 文件加载（带缓存） |
| `save_plugin_data` | `(plugin_name) → bool` | 将缓存写入 JSON 文件 |
| `get_plugin_data` | `(plugin_name) → Dict` | 获取内存缓存中的数据 |
| `clear_plugin_data` | `(plugin_name) → bool` | 删除文件并清除缓存 |

#### 存储规则

- **存储路径**: `data/[plugin_name].json`
- **缓存策略**: 内存缓存 `_data_cache: Dict[str, Dict]`，首次访问时惰性加载
- **自动保存**: 服务关闭时保存所有缓存数据

#### 错误处理

- 文件不存在 → 返回空字典（DEBUG 级别日志）
- JSON 解析失败 → 返回空字典（ERROR 级别日志）
- 其他异常 → 返回空字典（ERROR 级别日志）

### 3.5 RBACService — 角色权限控制服务

**文件**: `ncatbot/core/service/builtin/rbac/`

提供基于角色的层级权限控制（Role-Based Access Control），支持权限前缀树、角色继承、黑白名单。

#### 核心组件

##### PermissionTrie — 权限前缀树

以树结构高效存储和匹配权限路径，支持通配符：

| 通配符 | 含义 | 示例 |
|--------|------|------|
| `*` | 匹配当前层级 | `plugin.*` 匹配 `plugin.foo` |
| `**` | 匹配后续所有层级 | `plugin.**` 匹配 `plugin.foo.bar.baz` |

方法：`add(path)`、`remove(path)`、`exists(path, exact=False)`、`to_dict()`、`from_dict()`

##### PermissionChecker — 权限检查器

执行权限判定，遵循规则：**黑名单 > 白名单 > 默认拒绝**

```
check(user, permission) 流程：
    1. 收集用户直接权限（白名单 + 黑名单）
    2. 收集用户所有角色的权限（含继承链）
    3. 合并有效权限
    4. 黑名单命中 → 拒绝
    5. 白名单命中 → 允许
    6. 均未命中 → 默认拒绝
```

使用 LRU 缓存（256 条目）优化有效权限的计算。

##### EntityManager — 实体管理器

管理权限、角色和用户三类实体：

| 操作类别 | 方法 |
|----------|------|
| **权限管理** | `add_permission()`、`remove_permission()`、`permission_exists()` |
| **角色管理** | `add_role()`、`remove_role()`、`role_exists()`、`set_role_inheritance()` |
| **用户管理** | `add_user()`、`remove_user()`、`user_exists()`、`assign_role()`、`unassign_role()` |
| **权限分发** | `grant(target_type, target, permission, mode)`、`revoke(target_type, target, permission)` |

#### 角色继承

支持角色之间的继承关系，并内置循环继承检测：

```
admin → moderator → default
  │
  │  admin 自动继承 moderator 和 default 的所有白名单权限
  └─ 但黑名单始终优先
```

#### 内部数据结构

```python
_permissions: PermissionTrie              # 权限路径前缀树
_roles: Dict[str, Dict]                   # {role: {whitelist: set, blacklist: set}}
_users: Dict[str, Dict]                   # {user: {whitelist, blacklist, roles}}
_role_users: Dict[str, Set[str]]          # {role: {users}} 反向索引
_role_inheritance: Dict[str, List[str]]   # {role: [parent_roles]}
```

#### 配置参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `storage_path` | `data/rbac.json` | 持久化文件路径 |
| `default_role` | - | 自动创建的默认角色 |
| `case_sensitive` | `True` | 权限路径是否区分大小写 |

#### 关键方法

| 方法 | 签名 | 说明 |
|------|------|------|
| `add_permission` | `(path)` | 添加权限路径 |
| `grant` | `(target_type, target, permission, mode, create_permission)` | 向用户/角色授予权限 |
| `revoke` | `(target_type, target, permission)` | 撤销权限 |
| `check` | `(user, permission, create_user) → bool` | 检查用户是否具有权限 |
| `set_role_inheritance` | `(role, parent)` | 设置角色继承（含循环检测） |
| `save` | `()` | 持久化到 JSON 文件 |
| `_restore_state` | `()` | 从 JSON 文件恢复状态 |

### 3.6 FileWatcherService — 文件监控服务

**文件**: `ncatbot/core/service/builtin/file_watcher/service.py`

监控插件目录的文件变更，支撑热重载功能。通过后台守护线程轮询文件修改时间实现。

#### 工作机制

```
后台守护线程 (_watch_loop)
    │
    │  以 watch_interval 为间隔轮询
    │
    ├── 扫描 _watch_dirs 中所有文件
    │   │
    │   ├── 比对 _file_cache 中的修改时间
    │   │
    │   └── 发现变更 → 加入 _pending_dirs
    │
    ├── debounce_delay 后处理 _pending_dirs
    │
    └── 发布事件: ncatbot.plugin_file_changed
```

#### 关键方法

| 方法 / 属性 | 说明 |
|-------------|------|
| `add_watch_dir(directory)` | 添加目录到监控列表 |
| `pause()` | 暂停事件发布（线程仍在扫描） |
| `resume()` | 恢复事件发布 |
| `is_watching` | 线程是否在运行 |
| `pending_count` | 待处理的目录数量 |

#### 内部状态

```python
_file_cache: Dict[str, float]   # 文件路径 → 修改时间
_watch_dirs: Set[str]           # 监控的绝对路径集合
_pending_dirs: Set[str]         # 有变更待处理的目录
_paused: threading.Event        # 暂停控制标志
```

#### 配置参数

| 参数 | 默认值 | 测试模式 | 说明 |
|------|--------|----------|------|
| `watch_interval` | `1.0s` | `0.02s` | 轮询间隔 |
| `debounce_delay` | `1.0s` | `0.02s` | 防抖延迟 |

> 自动检测 `service_manager._test_mode`，测试模式下使用更短的间隔。

### 3.7 TimeTaskService — 定时任务服务

**文件**: `ncatbot/core/service/builtin/time_task/`

基于 `schedule` 库的定时任务调度服务，支持多种时间格式，通过事件总线驱动任务执行。

#### 支持的时间格式

| 类型 | 格式 | 示例 |
|------|------|------|
| **一次性任务** | `YYYY-MM-DD HH:MM:SS` | `2024-12-25 10:00:00` |
| **一次性任务** | `YYYY:MM:DD-HH:MM:SS` | `2024:12:25-10:00:00`（GitHub Actions 格式） |
| **每日任务** | `HH:MM` | `08:30` |
| **间隔任务** | 带单位 | `120s`、`2h30m`、`0.5d` |
| **间隔任务** | 冒号格式 | `00:15:30`（HH:MM:SS） |
| **间隔任务** | 中文自然语言 | `2天3小时5秒` |

单位转换：`d` = 86400s，`h` = 3600s，`m` = 60s，`s` = 1s

#### 关键方法

| 方法 | 签名 | 说明 |
|------|------|------|
| `add_job` | `(name, interval, conditions, max_runs, plugin_name) → bool` | 添加定时任务 |
| `remove_job` | `(name) → bool` | 移除定时任务 |

#### 参数说明

| 参数 | 类型 | 说明 |
|------|------|------|
| `name` | `str` | 任务唯一标识 |
| `interval` | `str` | 时间参数（支持上述所有格式） |
| `conditions` | `List[Callable]` | 条件列表（全部为 True 才执行） |
| `max_runs` | `Optional[int]` | 最大执行次数限制 |
| `plugin_name` | `str` | 关联插件名 |

#### 任务执行流程

```
定时器触发
    │
    ├── 检查 conditions（所有条件谓词）
    │   ├── 任一 False → 跳过
    │   └── 全部 True → 继续
    │
    ├── 检查 max_runs（执行次数限制）
    │
    └── 发布事件: ncatbot.[plugin_name].[task_name]
            │
            └── 订阅者处理实际业务逻辑
```

#### 配置参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `tick_interval` | `0.2s` | 调度器检查间隔 |

#### 内部结构

```python
_scheduler: Scheduler          # schedule 库调度器
_jobs: List[Dict]              # 任务元数据列表
_lock: threading.RLock         # 线程安全的可重入锁
```

使用后台守护线程运行调度循环。

### 3.8 UnifiedRegistryService — 统一注册服务

**文件**: `ncatbot/core/service/builtin/unified_registry/`

集中管理命令、过滤器和事件处理器的注册与执行，是插件系统与事件系统之间的桥梁。

#### 三大注册表

| 注册表 | 功能 |
|--------|------|
| `command_registry` | 命令定义、分组和执行 |
| `filter_registry` | 函数式过滤器模式 |
| `event_registry` | 事件处理器回调（通知、请求、元事件） |

#### 事件处理方法

| 方法 | 签名 | 说明 |
|------|------|------|
| `handle_message_event` | `(event)` | 执行命令匹配和分发 + 纯过滤器 |
| `handle_notice_event` | `(event) → bool` | 执行所有通知处理器 |
| `handle_request_event` | `(event) → bool` | 执行所有请求处理器 |
| `handle_meta_event` | `(event) → bool` | 执行所有元事件处理器 |

#### 注册方法

| 方法 | 说明 |
|------|------|
| `register_notice_handler(func)` | 装饰器，注册通知处理器 |
| `register_request_handler(func)` | 装饰器，注册请求处理器 |
| `set_current_plugin_name(name)` | 设置当前注册上下文的插件名 |

#### 插件生命周期集成

| 方法 | 触发时机 | 说明 |
|------|----------|------|
| `handle_plugin_load()` | 插件加载后 | 重新初始化注册表 |
| `handle_plugin_unload(plugin_name)` | 插件卸载时 | 清理该插件注册的条目 |

#### 内部组件

**FunctionExecutor**：函数执行器
- 自动注入 `bot_client` 或插件实例到函数参数
- 支持上下文感知的依赖注入

**CommandRunner**：命令运行器
- 前缀匹配
- 参数解析
- 权限检查（通过 RBAC 服务，如果可用）

## 5. 自定义服务开发

### 创建自定义服务

```python
from ncatbot.core.service import BaseService


class MyCustomService(BaseService):
    name = "my_service"
    description = "自定义服务示例"

    def __init__(self, **config):
        super().__init__(**config)
        self._connection = None

    async def on_load(self):
        """加载时建立连接"""
        host = self.config.get("host", "localhost")
        port = self.config.get("port", 8080)
        # 建立连接...
        self._connection = await self._connect(host, port)

    async def on_close(self):
        """关闭时释放连接"""
        if self._connection:
            await self._connection.close()
            self._connection = None

    async def _connect(self, host, port):
        # 实际连接逻辑
        pass

    def do_something(self):
        """对外提供的业务方法"""
        if not self.is_loaded:
            raise RuntimeError("服务未加载")
        # 业务逻辑...
```

### 注册和使用

```python
from ncatbot.core.service import ServiceManager

manager = ServiceManager()

# 注册时可传入配置
manager.register(MyCustomService, host="127.0.0.1", port=9090)

# 加载
await manager.load("my_service")

# 使用
service = manager.get("my_service")
service.do_something()

# 卸载
await manager.unload("my_service")
```

### 访问其他服务

服务实例加载时会自动注入 `service_manager`，可以通过它访问其他已加载的服务：

```python
class MyService(BaseService):
    name = "my_service"

    async def on_load(self):
        # 访问消息路由服务
        router = self.service_manager.message_router
        if router:
            result = await router.send("get_login_info", {})

        # 访问事件总线
        if self.event_bus:
            self.event_bus.subscribe("my_event", self._handle_event)
```

## 6. 设计要点

### 依赖注入

每个服务在加载时由 ServiceManager 注入 `service_manager` 引用，服务间通过该引用相互访问，避免了硬编码耦合。

### 生命周期保障

- `_load()`/`_close()` 内置防重复机制（检查 `_loaded` 标志）
- `close_all()` 确保所有服务正常关闭，避免资源泄漏

### 线程安全

- **MessageRouter**: `threading.Lock` 保护 Future 字典
- **RBACService**: LRU 缓存 + 原子操作
- **FileWatcherService / TimeTaskService**: 后台守护线程 + 同步原语
- **PluginDataService**: 内存缓存隔离写入

### 持久化策略分层

| 层级 | 服务 | 格式 | 特点 |
|------|------|------|------|
| 配置层 | PluginConfigService | YAML | 延迟保存 / 原子保存双模式 |
| 数据层 | PluginDataService | JSON | 内存缓存 + 惰性加载 |
| 权限层 | RBACService | JSON | 状态快照 + 恢复 |
