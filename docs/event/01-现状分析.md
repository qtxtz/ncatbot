# Event 模块现状分析

## 1. 文件结构

```
core/event/
├── __init__.py           # 汇总导出（star import 链）
├── enums.py              # 枚举：PostType, MessageType, NoticeType, ...
├── models.py             # 数据模型：BaseSender, GroupSender, Anonymous, FileInfo, Status
├── context.py            # ContextMixin：注入 api / services 到事件对象
├── mixins.py             # 行为 Mixin：MessageActionMixin, GroupAdminMixin, RequestActionMixin
├── events.py             # 事件类层次：BaseEvent → MessageEvent / NoticeEvent / ...
├── parser.py             # EventParser：(post_type, secondary_key) → 事件类 的注册表
└── message_segments/
    ├── __init__.py
    ├── base.py           # MessageSegment 基类 + TYPE_MAP 自动注册 + MessageArrayDTO
    ├── primitives.py     # PlainText, Face, At, Reply
    ├── media.py          # DownloadableMessageSegment, Image, Record, Video, File
    ├── misc.py           # Share, Location, Music, Json, Markdown
    ├── forward.py        # Forward, Node（嵌套转发消息）
    └── message_array.py  # MessageArray（构造/解析接口）+ CQ 码解析 + Text/AtAll 兼容类
```

### 2.4 辅助类

```
NcatBotEvent          EventBus 内部传输信封：type + data + _results + _propagation_stopped

Text / AtAll          4.4.x 兼容 dataclass（非 MessageSegment 子类）
```

## 3. 数据流

```
Adapter 收到 raw dict
  → EventDispatcher.dispatch(data)
    → EventParser.parse(data, api)        # dict → BaseEvent 子类 + bind_api
      → event.bind_services(services)     # 注入 ServiceManager
    → NcatBotEvent(type_str, event)       # 包装为 EventBus 信封
    → EventBus.publish(ncatbot_event)
      → HandlerDispatcher._on_event(ncatbot_event)
        → handler(ncatbot_event)          # handler 拿到的是 NcatBotEvent, .data 是 BaseEvent
```

## 4. 问题清单

### P1: 职责混杂 — 事件既是数据又是行为载体

`BaseEvent` 继承 `ContextMixin`，每个事件对象持有 `_api` 和 `_services` 引用。
`MessageActionMixin` / `GroupAdminMixin` / `RequestActionMixin` 将 API 调用方法（`reply`, `kick`, `ban`, `approve`, `reject`）直接挂在事件类上。

**影响**:
- 事件对象不是纯数据，无法安全序列化/反序列化
- 事件类与 API 实现强耦合
- 无法在不注入 API 的情况下构造事件对象（测试困难）
- `ContextMixin` 作为 `BaseModel` 子类，`_api`/`_services` 用 `PrivateAttr`，Pydantic 行为复杂

### P2: Mixin 多继承导致字段冲突

`GroupMessageEvent` 同时继承 `MessageEvent` 和 `GroupAdminMixin`。两者都声明 `group_id` 和 `user_id`，依赖 MRO 解析。
`GroupAdminMixin` 的 `group_id`/`user_id` 类型是 `Any`（为了兼容），而 `MessageEvent` 的是 `Optional[str]` / `str`，类型不一致。

### P3: `field_validator` 散布，重复代码多

几乎每个事件子类都重复写 `@field_validator("xxx_id", mode="before") def _force_str(cls, v): return str(v)`。
`BaseDataModel` 试图用通用 validator 自动转换 `*_id` 字段，但只用于 `models.py` 中的 Sender 类，事件类不继承它。

### P4: `MessageSegment.type` 是 `ClassVar` 但参与序列化

`type` 声明为 `ClassVar[str]`，Pydantic 不会将其包含在 schema 中。
但 `to_dict()` 手动引用 `self.type`，`from_dict()` 从 `data["type"]` 查 TYPE_MAP。
`Music` 和 `Image` 的 `from_dict`/`to_dict` 需要手动重命名字段（`platform` ↔ `type`, `image_type` ↔ `type`），容易出错。

### P5: `MessageArray` 职责过重

`MessageArray` 同时承担：
- 消息段列表容器
- CQ 码解析（`parse_cq_code_to_onebot11`）
- 构造 API（`add_text`, `add_image`, `add_at`, `add_reply`）
- 过滤/查询 API（`filter`, `filter_text`, `concatenate_text`, `is_user_at`）
- 兼容旧版 `Text` / `AtAll` dataclass

### P6: `EventParser` 注册时机隐式

`register_builtin_events()` 在模块导入时自动执行。扩展性差——第三方无法注册自定义事件类型（需修改源码）。

### P7: `NcatBotEvent` 与 `BaseEvent` 概念重叠

`NcatBotEvent` 是 EventBus 的内部信封，`BaseEvent` 是 OneBot 数据。两层包装导致 handler 中需要 `event.data` 解包。
`NcatBotEvent.data` 返回 `copy(self._data)`，每次访问产生拷贝开销。

### P8: 导出混乱

`__init__.py` 使用多层 `from x import *` + `__all__.extend()`，IDE 提示不稳定。
`MessageEvent as MessageSentEvent` 是错误的别名（`MessageSentEvent` 应该有 `post_type="message_sent"`）。

### P9: Forward 嵌套处理复杂

`Forward.from_dict()` 需要处理省略数据（`[...]`）、消息事件格式、Node 格式等多种输入，逻辑复杂且脆弱。
`Node.to_node_dict()` 中需要判断 `Forward` 类型并递归展开，耦合了序列化逻辑。

## 5. 外部使用者

| 使用者 | 引用的类型 |
|--------|-----------|
| 插件开发者 | `MessageEvent`, `GroupMessageEvent`, `PrivateMessageEvent`, `MessageArray`, `Image`, `At`, `Reply`, `PlainText` |
| `EventDispatcher` | `BaseEvent`, `EventParser`, `NcatBotEvent` |
| `HandlerDispatcher` | `NcatBotEvent`（信封），handler 通过 `.data` 得到 `BaseEvent` |
| `CommandRunner` / `FunctionExecutor` | `MessageEvent` |
| API 模块 | `GroupMessageEvent`, `PrivateMessageEvent`, `Forward`, `MessageArray` |
| 过滤器系统 | `MessageEvent`, `BaseEvent` |
| `RegistryEngine` (legacy) | `MessageEvent`, `BaseEvent` |
