# 适配器与网络通信

适配器层是 NcatBot 与底层 QQ 协议（NapCat/OneBot11）之间的桥梁。它封装了 NapCat 服务的安装、启动、配置、登录认证以及 WebSocket 通信的全部细节，为上层提供透明的消息收发能力。

## 目录结构

```
ncatbot/core/adapter/
└── nc/
    ├── __init__.py            # 模块导出
    ├── service.py             # NapCatService — 主编排器
    ├── websocket.py           # NapCatWebSocket — 底层 WebSocket 连接
    ├── config_manager.py      # ConfigManager — OneBot11/WebUI 配置
    ├── auth.py                # AuthHandler — 登录认证
    ├── platform.py            # PlatformOps — 跨平台抽象
    └── utils.py               # 工具函数（下载、版本检查）
```

## 架构总览

```
┌──────────────────────────────────────────────────┐
```

## 1. NapCatService — 主编排器

**文件**: `ncatbot/core/adapter/nc/service.py`

NapCatService 是适配器层的入口，负责根据配置选择运行模式，并协调其他组件完成 NapCat 的完整启动流程。

### 运行模式

| 模式 | 配置 | 说明 |
|------|------|------|
| **本地模式** | `remote_mode = False` | 自动下载、安装、配置并启动本地 NapCat 实例 |
| **远端模式** | `remote_mode = True` | 连接到外部已运行的 NapCat 服务，仅做连通性检查 |

### 入口方法

```python
def launch(self) -> None
```

根据 `ncatbot_config.napcat.remote_mode` 路由到对应的启动流程。

便捷函数：

```python
def launch_napcat_service() -> None:
    NapCatService().launch()
```

### 本地模式启动流程

```
_launch_local()
    │
    ├── 1. 检查服务是否已运行 (is_service_ok)
    │   └── 已运行 → _check_remote_service() → 返回
    │
    ├── 2. 检测平台 (self.platform)
    │   └── 不支持 → 抛出 UnsupportedPlatformError
    │
    ├── 3. 安装/更新 (platform.check_and_update)
    │   └── 失败 → 抛出 NcatBotError
    │
    ├── 4. 配置 (config.configure_all)
    │   ├── OneBot11 WebSocket 配置
    │   ├── WebUI 配置
    │   └── 快速登录脚本
    │
    ├── 5. 启动 (platform.start_napcat)
    │
    └── 6. 处理登录 (_handle_login)
        ├── WebUI 启用 → 尝试快连接 → 失败则 AuthHandler 登录
        └── WebUI 禁用 → 等待 WebSocket 服务就绪
```

### 远端模式启动流程

```
_launch_remote()
    │
    ├── 1. 检查 WebSocket 连通性
    │
    └── 2. _check_remote_service()
        ├── 无 WebUI → 跳过详细检查
        └── 有 WebUI → AuthHandler.report_status()
            ├── OK → 正常
            ├── UIN_MISMATCH → 抛出 NcatBotLoginError
            └── ABNORMAL → 抛出 NcatBotLoginError
```

### WebSocket 连通性检测

```python
@staticmethod
async def _test_websocket_async(report_status: bool = False) -> bool
```

- 使用 `ncatbot_config.get_uri_with_token()` 获取 WebSocket URI
- 连接超时：5 秒
- 检查首条消息是否包含 `status: failed`（错误码 1403 = Token 错误）

```python
def is_service_ok(self, timeout: int = 0, show_info: bool = True) -> bool
```

- `timeout=0`：即时检查，立即返回
- `timeout>0`：每 0.5 秒轮询，直到超时或成功

```python
def wait_for_service(self, timeout: int = 60) -> None
```

- 默认等待 60 秒
- 超时抛出 `NcatBotError("连接 NapCat WebSocket 服务器超时")`

### 异常类型

| 异常 | 说明 |
|------|------|
| `NcatBotLoginError` | 登录相关错误（非致命） |
| `NcatBotError` | 服务异常通用错误 |

## 2. NapCatWebSocket — WebSocket 连接管理

**文件**: `ncatbot/core/adapter/nc/websocket.py`

这是最底层的 WebSocket 组件，只负责连接管理和原始消息收发，不涉及消息路由或业务逻辑。

### 类定义

```python
class NapCatWebSocket:
    def __init__(
        self,
        uri: str,
        message_callback: Optional[Callable[[dict], Awaitable[None]]] = None,
    ):
        self._uri = uri
        self._client: Optional[websockets.ClientConnection] = None
        self._message_callback = message_callback
        self._running = False
        self._listen_task: Optional[asyncio.Task] = None
        self._send_lock = asyncio.Lock()
        self._loop: Optional[asyncio.AbstractEventLoop] = None
```

### 连接管理

| 方法 | 说明 |
|------|------|
| `connect()` | 建立 WebSocket 连接（close_timeout=0.2, max_size=1GB, open_timeout=1s） |
| `disconnect()` | 优雅关闭连接（停止监听、取消任务、关闭客户端） |
| `connected` | 属性，是否已连接 |
| `uri` | 属性，当前连接 URI |

### 消息收发

```python
async def send(self, data: dict) -> None
```

- 使用 `asyncio.Lock` 保护并发发送
- 检查连接状态，未连接抛出 `ConnectionError`
- 序列化为 JSON 发送

```python
async def listen(self) -> None
```

- 阻塞式持续接收 WebSocket 消息
- 接收到消息后解析 JSON，调用 `_message_callback`
- 连接断开时：
  - 如果 `_running=False`（主动断开）→ 退出
  - 否则尝试 `_reconnect()`，失败抛出 `NcatBotConnectionError`

```python
def start_listening(self) -> asyncio.Task
```

- 非阻塞版本，创建异步任务返回

### 自动重连

```python
async def _reconnect(self, uri: str) -> bool
```

1. 检查 `NapCatService().is_service_ok()` 等待服务就绪
2. 尝试重新建立 WebSocket 连接
3. 成功返回 `True`，失败返回 `False`

### 回调机制

```python
def set_message_callback(self, callback: Callable[[dict], Awaitable[None]]) -> None
```

运行时可动态更新消息回调。回调接收解析后的 JSON 字典。

### 设计特点

- **职责分离**：只管连接和传输，不解析消息语义
- **协程安全**：`asyncio.Lock` 保护并发发送
- **回调驱动**：无消息队列，直接异步回调
- **自动重连**：连接断开后自动尝试恢复
- **任务化监听**：可作为独立异步任务运行

## 3. ConfigManager — 配置管理

**文件**: `ncatbot/core/adapter/nc/config_manager.py`

负责管理 NapCat 的 OneBot11 和 WebUI JSON 配置文件。

### 构造与属性

```python
class ConfigManager:
    def __init__(self, platform_ops: Optional[PlatformOps] = None):
        self._platform = platform_ops or PlatformOps.create()
        self._napcat_dir = self._platform.napcat_dir
        self._config_dir = self._platform.config_dir
```

| 属性 | 路径 |
|------|------|
| `uin` | `str(ncatbot_config.bot_uin)` |
| `onebot_config_path` | `{config_dir}/onebot11_{uin}.json` |
| `webui_config_path` | `{config_dir}/webui.json` |
| `quick_login_script_path` | `{napcat_dir}/{uin}_quickLogin.bat` |

### OneBot11 配置

```python
def configure_onebot(self) -> None
```

配置 WebSocket Server：

```python
# 生成的配置结构
{
    "name": "WsServer",
    "enable": True,
    "host": ncatbot_config.napcat.ws_listen_ip,
    "port": 3001,  # 从 ws_uri 解析
    "messagePostFormat": "array",
    "reportSelfMessage": False,
    "token": ncatbot_config.napcat.ws_token or "",
    "enableForcePushEvent": True,
    "debug": False,
    "heartInterval": 30000,
}
```

**端口冲突处理**：

1. 读取现有配置中的 `websocketServers` 数组
2. 检查是否已存在相同配置
3. 发现端口冲突时提示用户确认（y/n）
4. 用户拒绝则抛出 `ValueError`

### WebUI 配置

```python
def configure_webui(self) -> None
```

- **首次运行**：从默认模板创建，设置端口、Token、监听 IP
- **已有配置**：比对 `token`、`port`、`wsListenIp`，如有差异则更新并记录日志

### 快速登录脚本

```python
def configure_quick_login(self) -> None
```

- 仅 Windows 平台
- 从 `quickLoginExample.bat` 复制为 `{uin}_quickLogin.bat`

### 总配置入口

```python
def configure_all(self) -> None
```

按顺序执行：`configure_onebot()` → `configure_quick_login()` → `configure_webui()`

便捷函数：

```python
def config_napcat() -> None:
    ConfigManager().configure_all()
```

## 4. AuthHandler — 登录认证

**文件**: `ncatbot/core/adapter/nc/auth.py`

处理 QQ 登录的完整流程，包括 WebUI 认证、快速登录和二维码登录。

### 登录状态枚举

```python
class LoginStatus(IntEnum):
    OK = 0              # 已正确登录
    NOT_LOGGED_IN = 1   # 未登录
    UIN_MISMATCH = 2    # QQ 号不匹配
    ABNORMAL = 3        # 状态异常，需重启
```

### 异常类型

| 异常 | 说明 |
|------|------|
| `AuthError` | 认证基础异常 |
| `WebUIConnectionError` | WebUI 连接失败 |
| `LoginTimeoutError` | 登录超时 |
| `QQLoginedError` | QQ 已登录（内部使用） |

### 超时常量

| 常量 | 值 | 说明 |
|------|------|------|
| `WEBUI_CONNECT_TIMEOUT` | 90s | WebUI 连接超时 |
| `QRCODE_LOGIN_TIMEOUT` | 60s | 二维码登录超时 |
| `QRCODE_FETCH_TIMEOUT` | 15s | 获取二维码超时 |


### 登录状态检查

```python
def report_status(self) -> LoginStatus
```

综合检查逻辑：

```
1. check_login_status() → 未登录 → NOT_LOGGED_IN
2. get_login_info() → 获取失败 → ABNORMAL
3. info.uin != bot_uin → UIN_MISMATCH
4. info.online == False → ABNORMAL
5. 全部通过 → OK
```

### 登录流程

```python
def login(self) -> None
```

主登录入口，包含降级策略：

```
login()
    │
    ├── 1. report_status()
    │   ├── OK → 已登录，直接返回
    │   └── ABNORMAL/UIN_MISMATCH → 抛出 AuthError
    │
    ├── 2. 尝试快速登录 (quick_login)
    │   ├── 获取 quick_login_list
    │   ├── 目标 UIN 在列表中 → 调用 SetQuickLogin API
    │   └── 成功 → 返回
    │
    └── 3. 降级到二维码登录 (qrcode_login)
        ├── 获取二维码 URL
        ├── 终端显示二维码（ASCII Art）
        ├── 轮询等待扫码（60 秒超时，30 秒时警告）
        └── 验证最终状态
```

### 快速登录

```python
def quick_login(self) -> bool
```

1. 调用 `/api/QQLogin/GetQuickLoginListNew` 获取可快速登录的 QQ 列表
2. 筛选 `isQuickLogin == True` 的条目
3. 目标 UIN 在列表中 → 调用 `/api/QQLogin/SetQuickLogin`
4. 返回是否成功

### 二维码登录

```python
def qrcode_login(self) -> None
```

1. 调用 `/api/QQLogin/GetQQLoginQrcode` 获取二维码 URL
2. 使用 `qrcode` 库在终端渲染 ASCII Art 二维码
3. 轮询 `check_login_status()` 等待用户扫码，最长 60 秒
4. 30 秒无响应时输出警告

## 5. PlatformOps — 跨平台抽象

**文件**: `ncatbot/core/adapter/nc/platform.py`

通过工厂模式和抽象基类实现 Windows/Linux 双平台支持。

### 工厂方法

```python
@staticmethod
def create() -> "PlatformOps":
    system = platform.system()
    if system == "Windows":
        return WindowsOps()
    elif system == "Linux":
        return LinuxOps()
    else:
        raise UnsupportedPlatformError(system)
```

### 抽象接口

| 成员 | 类型 | 说明 |
|------|------|------|
| `napcat_dir` | 抽象属性 | NapCat 安装目录 |
| `config_dir` | 具体属性 | `{napcat_dir}/config` |
| `is_napcat_running(uin)` | 抽象方法 | 检查进程是否运行 |
| `install_napcat(install_type)` | 抽象方法 | 安装或更新 NapCat |
| `start_napcat(uin)` | 抽象方法 | 启动 NapCat |
| `stop_napcat()` | 抽象方法 | 停止 NapCat |

### 公共方法

```python
def check_and_update(self) -> bool
```

安装/更新编排器：

1. 未安装 → 调用 `install_napcat("install")`
2. 更新检查已禁用 → 返回
3. 比对本地版本和 GitHub 最新版本
4. 版本不同 → 调用 `install_napcat("update")`

版本检查 API：`https://api.github.com/repos/NapNeko/NapCatQQ/tags`

### WindowsOps 实现

| 功能 | 实现细节 |
|------|----------|
| **安装** | 从 GitHub Releases 下载 `NapCat.Shell.zip`，解压到 `WINDOWS_NAPCAT_DIR` |
| **启动** | 子进程执行 `{launcher}.bat {uin}`，输出重定向到 DEVNULL |
| **停止** | `taskkill /F /IM QQ.exe` |
| **启动器选择** | 根据 Windows 版本自动选择启动脚本 |

#### 启动器自动选择

```
Windows 11 → launcher.bat
Windows 10 → launcher-win10.bat
Server 2016/2019/2022 → launcher-win10.bat
Server 2025 → launcher.bat
```

### LinuxOps 实现

| 功能 | 实现细节 |
|------|----------|
| **安装** | 下载一键安装脚本，通过 `sudo bash` 执行 |
| **启动** | `sudo bash napcat start {uin}` |
| **停止** | `bash napcat stop` |
| **进程检测** | `bash napcat status`，解析输出中的 PID 信息 |

#### 特殊处理

- **Root 检查**：安装前验证 `sudo whoami == root`
- **多实例冲突**：启动时检测是否已有其他 UIN 在运行，提示用户选择
- **安装目录**：优先使用 `LINUX_NAPCAT_DIR`，备选 `~/Napcat/opt/QQ/...`

## 8. 关键数据流

### WebSocket 通信完整流程

```
NapCatService.launch()
    │
    ▼
[NapCat 进程启动]
    │
    ▼
MessageRouter.on_load()
    │
    ├── 创建 NapCatWebSocket(uri, callback)
    ├── websocket.connect()
    └── websocket.start_listening()
            │
            ▼
        [监听循环]
            │
            ├── 收到消息 (JSON)
            │   │
            │   ├── 有 echo 字段 → API 响应 → 设置 Future 结果
            │   │
            │   └── 无 echo 字段 → 事件推送 → EventDispatcher
            │
            └── 连接断开
                │
                └── _reconnect()
                    ├── 检查 NapCat 服务状态
                    └── 重新连接 WebSocket
```

### 登录认证流程

```
NapCatService._handle_login()
    │
    ├── WebUI 启用
    │   │
    │   ├── 尝试快速连接（3 秒）
    │   │   ├── 成功 → "快速登录成功"
    │   │   └── 失败 ↓
    │   │
    │   └── AuthHandler().login()
    │       │
    │       ├── _connect_webui()
    │       │   └── SHA256 Token 认证（最长 90 秒）
    │       │
    │       ├── report_status()
    │       │   └── OK / NOT_LOGGED_IN / UIN_MISMATCH / ABNORMAL
    │       │
    │       ├── quick_login()
    │       │   └── 调用 WebUI API 快速登录
    │       │
    │       └── qrcode_login()（降级方案）
    │           ├── 获取二维码 URL
    │           ├── 终端渲染二维码
    │           └── 轮询等待扫码（60 秒超时）
    │
    └── WebUI 禁用
        └── wait_for_service(websocket_timeout)
```

## 10. 设计要点

### 工厂 + 抽象基类（平台适配）

通过 `PlatformOps.create()` 工厂方法根据运行环境自动创建 `WindowsOps` 或 `LinuxOps`，上层代码无需关心底层平台差异。

### 分层职责

- **NapCatService**：编排器，不直接操作进程或网络
- **PlatformOps**：进程管理（安装、启动、停止）
- **ConfigManager**：配置文件读写
- **AuthHandler**：认证逻辑
- **NapCatWebSocket**：纯网络通信

### 优雅降级

登录流程具备多级降级策略：快速登录 → 二维码登录 → 超时报错。每一级失败后自动尝试下一级，最大限度保证可用性。

### 自动重连

WebSocket 连接断开后自动检测服务状态并尝试重连，无需上层干预。
