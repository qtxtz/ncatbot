# 事件系统

本文档详细介绍 NcatBot 的事件系统，包括事件类型层级、数据模型、事件解析器、上下文管理和操作混入。

> 源码位置：`ncatbot/core/event/`

## 1. 事件类型层级

NcatBot 的事件体系基于 OneBot v11 协议，使用 Pydantic v2 模型定义，所有事件类都继承自 `BaseEvent`。

### 1.1 完整类型树

```
BaseEvent (ContextMixin)
│
├── MessageEvent (MessageActionMixin)
│   ├── PrivateMessageEvent         — 私聊消息
│   └── GroupMessageEvent           — 群聊消息 (GroupAdminMixin)
│
├── NoticeEvent
│   ├── GroupUploadNoticeEvent      — 群文件上传
│   ├── GroupAdminNoticeEvent       — 群管理员变动
│   ├── GroupDecreaseNoticeEvent    — 群成员减少
│   ├── GroupIncreaseNoticeEvent    — 群成员增加 (GroupAdminMixin)
│   ├── GroupBanNoticeEvent         — 群禁言
│   ├── FriendAddNoticeEvent        — 好友添加
│   ├── GroupRecallNoticeEvent      — 群消息撤回
│   ├── FriendRecallNoticeEvent     — 好友消息撤回
│   └── NotifyEvent                 — 系统通知
│       ├── PokeNotifyEvent         — 戳一戳
│       ├── LuckyKingNotifyEvent    — 运气王
│       └── HonorNotifyEvent        — 群荣誉
│
├── RequestEvent (RequestActionMixin)
│   ├── FriendRequestEvent          — 好友申请
│   └── GroupRequestEvent           — 加群请求/邀请
│
└── MetaEvent
    ├── LifecycleMetaEvent          — 生命周期（连接/断开）
    └── HeartbeatMetaEvent          — 心跳
```

### 1.2 基础事件类

```python
class BaseEvent(ContextMixin):
    """所有事件的基类"""
    time: int                  # 事件发生的 Unix 时间戳
    self_id: str               # 机器人 QQ 号（自动转为字符串）
    post_type: PostType        # 事件大类：message / notice / request / meta_event
```

`ContextMixin` 提供依赖注入能力，使事件对象可以访问 `BotAPI` 实例。

## 2. 消息事件

### 2.1 MessageEvent

```python
class MessageEvent(BaseEvent, MessageActionMixin):
    """消息事件基类 — 私聊和群聊的公共字段"""
    message_type: MessageType       # private / group
    sub_type: str                   # friend / group / discuss / normal 等
    message_id: str                 # 消息 ID（自动转为字符串）
    user_id: str                    # 发送者 QQ 号（自动转为字符串）
    message: MessageArray           # 消息内容（消息段数组）
    raw_message: str                # 原始文本消息
    sender: BaseSender              # 发送者信息
    font: int = 0                   # 字体
```

### 2.2 PrivateMessageEvent

```python
class PrivateMessageEvent(MessageEvent):
    """私聊消息事件"""
    sub_type: str = "friend"        # friend（好友）/ group（群临时会话）/ discuss（讨论组）
```

### 2.3 GroupMessageEvent

```python
class GroupMessageEvent(MessageEvent, GroupAdminMixin):
    """群聊消息事件 — 额外包含群管理操作"""
    group_id: str                          # 群号（自动转为字符串）
    anonymous: Optional[Anonymous] = None  # 匿名信息（匿名消息时有值）
    sender: GroupSender                    # 群发送者信息（扩展字段）
```

### 2.4 MessageArray — 消息段数组

`message` 字段是一个 `MessageArray` 对象，包含一个或多个消息段（CQ 码的结构化表示）：

```python
# 消息段示例
[
    {"type": "text", "data": {"text": "你好 "}},
    {"type": "at", "data": {"qq": "123456"}},
    {"type": "image", "data": {"file": "xxx.jpg", "url": "https://..."}},
]
```

常见消息段类型：

| type | 说明 | data 字段 |
|------|------|----------|
| `text` | 纯文本 | `text` |
| `face` | QQ 表情 | `id` |
| `image` | 图片 | `file`, `url`, `type` |
| `record` | 语音 | `file`, `url` |
| `video` | 视频 | `file`, `url` |
| `at` | @某人 | `qq` |
| `reply` | 回复 | `id` |
| `forward` | 转发 | `id` |
| `json` | JSON 消息 | `data` |
| `xml` | XML 消息 | `data` |
| `file` | 文件 | `file`, `name`, `size` |

## 8. 事件解析器

> 源码位置：`ncatbot/core/event/parser.py`

`EventParser` 负责将原始 OneBot 数据字典转换为类型化的事件对象。

### 8.1 注册机制

事件类通过装饰器注册到解析器：

```python
# 内部注册示例
@EventParser.register("message", "group")
class GroupMessageEvent(MessageEvent):
    ...

@EventParser.register("notice", "group_increase")
class GroupIncreaseNoticeEvent(NoticeEvent):
    ...
```

注册键为 `(post_type, secondary_key)` 元组。

### 8.2 解析流程

```python
EventParser.parse(data: dict) -> BaseEvent
```

1. 从 `data` 中提取 `post_type`
2. 根据 `post_type` 确定二级键名（如 `message_type`、`notice_type`）
3. 查找注册表中的事件类
4. 使用 Pydantic 模型验证并创建事件实例
5. 注入 API 实例（通过 `bind_api()`）

### 8.3 自动类型转换

事件类中的字段验证器会自动处理类型转换：

```python
# 以下字段会自动转为 str，即使 OneBot 返回 int
message_id: str      # 123 → "123"
user_id: str         # 456 → "456"
group_id: str        # 789 → "789"
self_id: str         # 111 → "111"
```

## 9. 上下文管理

> 源码位置：`ncatbot/core/event/context.py`

### 9.1 ContextMixin

`ContextMixin` 是一个混入类，为事件对象提供 API 访问能力：

```python
class ContextMixin:
    """通过 bind_api() 注入 BotAPI 实例"""

    def bind_api(self, api: IBotAPI) -> None:
        """绑定 API 实例到事件对象"""
        self._api = api

    @property
    def api(self) -> IBotAPI:
        """获取已绑定的 API 实例"""
        return self._api
```

### 9.2 IBotAPI 协议

```python
class IBotAPI(Protocol):
    """API 接口协议，定义事件操作所需的最小接口"""
    async def post_group_msg(self, group_id, **kwargs) -> Any: ...
    async def post_private_msg(self, user_id, **kwargs) -> Any: ...
    async def delete_msg(self, message_id) -> Any: ...
    async def set_group_kick(self, group_id, user_id, **kwargs) -> Any: ...
    async def set_group_ban(self, group_id, user_id, **kwargs) -> Any: ...
    async def set_friend_add_request(self, flag, **kwargs) -> Any: ...
    async def set_group_add_request(self, flag, **kwargs) -> Any: ...
```

## 10. 操作混入

> 源码位置：`ncatbot/core/event/mixins.py`

操作混入为事件对象提供便捷的操作方法，通过 `ContextMixin` 注入的 API 实现。

### 10.1 MessageActionMixin — 消息操作

```python
class MessageActionMixin(ContextMixin):

    async def reply(self, text: str = "", **kwargs) -> Any:
        """回复消息 — 自动判断群聊/私聊"""
        if hasattr(self, "group_id"):
            return await self.api.post_group_msg(self.group_id, text=text, **kwargs)
        else:
            return await self.api.post_private_msg(self.user_id, text=text, **kwargs)

    async def delete(self) -> Any:
        """删除（撤回）本条消息"""
        return await self.api.delete_msg(self.message_id)
```

使用示例：
```python
@bot.on_group_message()
async def handler(msg: GroupMessageEvent):
    # 直接回复，不需要手动判断群聊/私聊
    await msg.reply(text="收到了！")

    # 撤回本条消息
    await msg.delete()
```

### 10.2 GroupAdminMixin — 群管理操作

```python
class GroupAdminMixin(ContextMixin):

    async def kick(self, reject_add_request: bool = False) -> Any:
        """踢出群成员"""
        return await self.api.set_group_kick(
            self.group_id, self.user_id,
            reject_add_request=reject_add_request
        )

    async def ban(self, duration: int = 30 * 60) -> Any:
        """禁言群成员（默认 30 分钟）"""
        return await self.api.set_group_ban(
            self.group_id, self.user_id,
            duration=duration
        )
```

使用示例：
```python
@bot.on_group_message()
async def handler(msg: GroupMessageEvent):
    if "违规内容" in msg.raw_message:
        await msg.delete()           # 撤回消息
        await msg.ban(duration=600)  # 禁言 10 分钟
```

### 10.3 RequestActionMixin — 请求处理

```python
class RequestActionMixin(ContextMixin):

    async def approve(self, remark: str = "", reason: str = "") -> Any:
        """同意请求"""
        if hasattr(self, "group_id"):
            return await self.api.set_group_add_request(
                self.flag, approve=True, reason=reason
            )
        else:
            return await self.api.set_friend_add_request(
                self.flag, approve=True, remark=remark
            )

    async def reject(self, reason: str = "") -> Any:
        """拒绝请求"""
        if hasattr(self, "group_id"):
            return await self.api.set_group_add_request(
                self.flag, approve=False, reason=reason
            )
        else:
            return await self.api.set_friend_add_request(
                self.flag, approve=False
            )
```

使用示例：
```python
@bot.on_request()
async def handler(event: FriendRequestEvent):
    # 自动同意好友申请
    await event.approve(remark="新好友")

@bot.on_request()
async def handler(event: GroupRequestEvent):
    if event.sub_type == "invite":
        await event.approve()    # 同意邀请
    else:
        await event.reject(reason="暂不开放")  # 拒绝加群
```

## 11. 事件处理完整流程示例

```python
# 1. WebSocket 接收到原始数据
raw_data = {
    "post_type": "message",
    "message_type": "group",
    "sub_type": "normal",
    "message_id": 12345,
    "group_id": 67890,
    "user_id": 11111,
    "message": [{"type": "text", "data": {"text": "你好"}}],
    "raw_message": "你好",
    "sender": {"user_id": 11111, "nickname": "张三", "role": "member"},
    "time": 1710000000,
    "self_id": 22222,
    "font": 0
}

# 2. EventDispatcher 分发
#    → EventParser 识别为 GroupMessageEvent
#    → 创建 GroupMessageEvent 实例
#    → bind_api(bot_api)
#    → 构建 NcatBotEvent(type="ncatbot.message.group", data=event)

# 3. EventBus 匹配处理器
#    → 精确匹配 "ncatbot.message.group" 的处理器
#    → 前缀匹配 "ncatbot.message" 的处理器
#    → 正则匹配 "re:ncatbot\.message\..*" 的处理器
#    → 按优先级排序执行

# 4. 处理器接收事件
@bot.on_group_message()
async def handler(msg: GroupMessageEvent):
    # msg.raw_message == "你好"
    # msg.group_id == "67890"
    # msg.user_id == "11111"
    # msg.sender.nickname == "张三"
    await msg.reply(text="你也好！")
    # → 调用 api.post_group_msg(group_id="67890", text="你也好！")
    # → MessageRouter.send("send_group_msg", {...})
    # → WebSocket.send({action, params, echo})
    # → 等待 echo 响应
```
