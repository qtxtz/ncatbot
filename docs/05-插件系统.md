# 插件系统

本文档详细介绍 NcatBot 的插件系统，包括插件基类、生命周期、加载器、依赖解析和权限控制等。

> 源码位置：`ncatbot/plugin_system/`

## 1. 架构概述

```
┌─────────────────────────────────────────────────┐
│                  PluginLoader                    │
│          (发现、索引、加载、卸载插件)              │
├────────────┬────────────┬───────────────────────┤
│ _ModuleImporter │ _DependencyResolver │ hooks   │
│  (模块导入)     │   (依赖解析)        │ (迁移)   │
└────────┬───────┴──────┬─────┴───────────────────┘
         │              │
         ▼              ▼
┌────────────────────────────────────────────────┐
│              NcatBotPlugin                      │
│       (BasePlugin + TimeTaskMixin              │
│        + ConfigMixin)                           │
├────────────────────────────────────────────────┤
│  framework injection:                          │
│    services: ServiceManager                    │
│    api: BotAPI                                 │
│    _event_bus: EventBus                        │
│    config: PluginConfig (持久化)                │
│    data: Dict (持久化)                          │
└────────────────────────────────────────────────┘
```

## 2. BasePlugin — 插件基类

> 源码位置：`ncatbot/plugin_system/base_plugin.py`

`BasePlugin` 定义了插件的元数据声明和生命周期接口。实际初始化逻辑由 `NcatBotPlugin`（内置混入）实现。

### 2.1 元数据声明

每个插件**必须**定义以下属性：

```python
class MyPlugin(BasePlugin):
    name = "my_plugin"               # 插件唯一标识符（必填）
    version = "1.0.0"               # 插件版本号（必填）
    author = "作者名"                # 作者（可选，默认 "Unknown"）
    description = "插件描述"         # 描述（可选）
    dependencies = {                 # 依赖声明（可选）
        "other_plugin": ">=1.0.0"   # 依赖插件名: 版本约束
    }
```

### 2.2 运行时属性

框架在加载插件时会自动注入以下属性：

| 属性 | 类型 | 说明 |
|------|------|------|
| `workspace` | `Path` | 插件工作目录（自动创建） |
| `config` | `PluginConfig` | 持久化配置（自动加载/保存） |
| `data` | `Dict[str, Any]` | 持久化数据（JSON 格式，自动加载/保存） |
| `services` | `ServiceManager` | 服务管理器引用 |
| `api` | `BotAPI` | Bot API 接口 |
| `_event_bus` | `EventBus` | 事件总线 |
| `_debug` | `bool` | 调试模式标志 |

### 2.3 便捷属性

```python
class BasePlugin:
    @property
    def event_bus(self) -> EventBus:
        """获取事件总线"""

    @property
    def debug(self) -> bool:
        """是否处于调试模式"""

    @property
    def meta_data(self) -> Dict[str, Any]:
        """获取插件元数据字典"""

    @property
    def rbac(self) -> RBACService:
        """获取 RBAC 权限服务"""

    @property
    def config_service(self) -> PluginConfigService:
        """获取配置服务"""
```

## 3. 插件生命周期

### 3.1 生命周期图

```
                  ┌──────────┐
                  │  发现索引  │  ← _ModuleImporter 解析 manifest.toml
                  └────┬─────┘
                       │
                  ┌────▼─────┐
                  │  依赖解析  │  ← _DependencyResolver 拓扑排序
                  └────┬─────┘
                       │
                  ┌────▼─────┐
                  │ __init__  │  ← 实例化插件类
                  └────┬─────┘
                       │
                  ┌────▼─────┐
                  │ _init_()  │  ← 同步初始化钩子（用户可选实现）
                  └────┬─────┘
                       │
                  ┌────▼──────┐
                  │__onload__ │  ← 框架内部：加载配置、数据、注入依赖
                  └────┬──────┘
                       │
                  ┌────▼─────┐
                  │ on_load() │  ← 异步初始化钩子（用户实现主要逻辑）
                  └────┬─────┘
                       │
                  ┌────▼─────┐
                  │  运行中    │  ← 处理事件、定时任务等
                  └────┬─────┘
                       │
              ┌────────┼────────┐
              ▼                 ▼
       ┌───────────┐     ┌───────────┐
       │  重载流程   │     │  卸载流程   │
       ├───────────┤     ├───────────┤
       │on_reload()│     │ _close_() │  ← 同步清理
       │_reinit_() │     │on_close() │  ← 异步清理
       └───────────┘     │__unload__│  ← 框架清理（注销处理器）
                         └───────────┘
```

### 3.2 生命周期钩子

```python
class MyPlugin(BasePlugin):
    name = "my_plugin"
    version = "1.0.0"

    def _init_(self):
        """同步初始化（在 on_load 之前调用）"""
        # 适合不需要异步的初始化逻辑
        self.counter = 0

    async def on_load(self):
        """异步初始化（主要初始化逻辑）"""
        # 注册事件处理器
        self.register_handler(
            "ncatbot.message.group",
            self.handle_message,
            priority=0
        )
        # 读取配置
        self.threshold = self.config.get("threshold", 10)

    async def on_reload(self):
        """重载时调用"""
        # 重新加载配置等
        self.threshold = self.config.get("threshold", 10)

    def _reinit_(self):
        """同步重初始化"""
        self.counter = 0

    async def on_close(self):
        """异步清理"""
        # 保存数据、释放资源等
        await self.save_data()

    def _close_(self):
        """同步清理"""
        ...
```

### 3.3 框架内部生命周期 (`__onload__` / `__unload__`)

这些方法由框架自动调用，不建议用户重写：

- **`__onload__`**：加载配置数据、注入 API 和 services、执行数据迁移
- **`__unload__`**：注销所有事件处理器、保存数据

## 4. 事件处理

### 4.1 注册事件处理器

```python
class MyPlugin(BasePlugin):
    async def on_load(self):
        # 方式一：register_handler
        handler_id = self.register_handler(
            event_type="ncatbot.message.group",
            handler=self.handle_group_msg,
            priority=10,              # 优先级
            timeout=30.0,             # 超时时间
        )

        # 方式二：注册所有类型的消息
        self.register_handler(
            "ncatbot.message",          # 前缀匹配，捕获所有消息
            self.handle_all_msgs
        )

    async def handle_group_msg(self, event: NcatBotEvent):
        msg = event.data  # GroupMessageEvent
        if msg.raw_message == "你好":
            await msg.reply(text="你也好！")
```

### 4.2 注销事件处理器

```python
# 注销单个处理器
self.unregister_handler(handler_id)

# 注销本插件的所有处理器
self.unregister_all_handler()
```

### 4.3 发布自定义事件

```python
# 发布事件供其他插件处理
results = await self.publish("my_plugin.custom_event", {"key": "value"})
```

## 5. 配置系统

### 5.1 PluginConfig

每个插件有独立的配置对象，支持持久化到 YAML 文件：

```python
class MyPlugin(BasePlugin):
    async def on_load(self):
        # 获取配置项（带默认值）
        threshold = self.config.get("threshold", 10)
        enabled = self.config.get("enabled", True)

        # 设置配置项
        self.config.set("threshold", 20)

        # 注册配置项（附带描述，用于配置管理UI）
        self.config_service.register_config(
            key="threshold",
            default=10,
            description="触发阈值"
        )
```

### 5.2 ConfigMixin

`ConfigMixin` 提供更便捷的配置注册方式：

```python
class MyPlugin(BasePlugin):
```

## 6. 数据持久化

### 6.1 data 字典

每个插件有一个 `data` 字典，自动持久化到 JSON 文件：

```python
class MyPlugin(BasePlugin):
    async def on_load(self):
        # data 在加载时自动从文件读取
        self.counter = self.data.get("counter", 0)

    async def handle_message(self, event):
        self.counter += 1
        self.data["counter"] = self.counter
        # data 在插件卸载时自动保存
```

### 6.2 工作目录

每个插件有独立的工作目录（`self.workspace`），位于 `data/<plugin_name>/` 下：

```python
class MyPlugin(BasePlugin):
    async def on_load(self):
        # 插件工作目录
        print(self.workspace)  # Path("data/my_plugin/")

        # 在工作目录中创建文件
        cache_file = self.workspace / "cache.json"
```

## 7. 定时任务

通过 `TimeTaskMixin` 混入，插件可以注册定时任务：

```python
class MyPlugin(BasePlugin):
    async def on_load(self):
        # 每隔 60 秒执行
        self.add_time_task(
            name="heartbeat",
            func=self.heartbeat,
            interval=60,           # 秒
        )

        # 每天定时执行
        self.add_scheduled_task(
            name="daily_report",
            func=self.daily_report,
            time="08:00",          # 每天 8:00
        )

    async def heartbeat(self):
        print("心跳检查...")

    async def daily_report(self):
        print("生成日报...")
```

## 8. 插件加载器

> 源码位置：`ncatbot/plugin_system/loader/`

### 8.1 PluginLoader

`PluginLoader` 管理插件的完整生命周期：

```python
class PluginLoader:
    def __init__(self, event_bus, service_manager, *, debug=False):
        self._importer = _ModuleImporter()      # 模块导入器
        self._resolver = _DependencyResolver()  # 依赖解析器
        self.plugins: Dict[str, BasePlugin] = {}  # 已加载插件
```

### 8.2 加载流程

```
load_external_plugins(path="plugins/")
    │
    ├── 1. _pre_first_load(path)
    │       ├── interactive_migrate_plugins()  ← 旧版数据迁移
    │       └── index_all_plugins(path)        ← 索引所有插件目录
    │
    ├── 2. _importer.get_plugin_manifests(path) ← 获取所有 manifest.toml
    │       返回: {name: {class, version, dependencies, ...}}
    │
    ├── 3. _resolver.build(manifests)          ← 构建依赖图
    │
    ├── 4. _resolver.resolve()                 ← 拓扑排序 + 循环检测
    │       返回: [name1, name2, ...]  (加载顺序)
    │
    ├── 5. for name in load_order:
    │       └── load_plugin(name)
    │           └── _load_plugin_by_class(plugin_class, name)
    │               ├── 实例化插件
    │               ├── 注入 services, api, event_bus
    │               ├── 设置 workspace
    │               ├── 调用 _init_()
    │               ├── 调用 __onload__()
    │               └── 调用 on_load()
    │
    └── 6. _validate_versions()                ← 验证依赖版本
```

### 8.3 插件目录结构

每个插件是一个独立目录，包含 `manifest.toml` 和 Python 源文件：

```
plugins/
└── my_plugin/
    ├── manifest.toml          # 插件清单（必需）
    ├── main.py                # 插件入口（必需，包含插件类）
    ├── requirements.txt       # 依赖包（可选，自动安装）
    └── ...                    # 其他文件
```

### 8.4 manifest.toml 格式

```toml
[plugin]
name = "my_plugin"
version = "1.0.0"
author = "作者名"
description = "插件描述"
entry = "main.py"              # 入口文件（默认 main.py）

[dependencies]
other_plugin = ">=1.0.0"       # 依赖其他插件
```

> 源码位置：`ncatbot/plugin_system/loader/resolver.py`

```python
        # 3. 返回有序的插件名称列表

```

### 10.2 依赖处理示例

```
插件 A 依赖 B >=1.0.0
插件 B 依赖 C
插件 D 无依赖

构建依赖图：
A → B → C
D (独立)

拓扑排序结果：[C, B, A, D] 或 [D, C, B, A]
即被依赖的插件先加载
```

### 10.3 错误处理

- **循环依赖**：`A → B → C → A` 会抛出异常并提示依赖链
- **缺失依赖**：依赖的插件不存在时发出警告
- **版本不满足**：版本约束不满足时发出警告

## 13. RBAC 权限控制

> 源码位置：`ncatbot/core/service/builtin/rbac/`

### 13.1 概念

| 概念 | 说明 | 示例 |
|------|------|------|
| **权限** | 点分隔的权限路径 | `chat.group.send`、`admin.kick` |
| **角色** | 权限集合 | `admin`、`moderator`、`user` |
| **通配符** | 匹配任意子路径 | `chat.group.*` |
| **黑名单** | 显式拒绝（优先级最高） | — |
| **白名单** | 显式允许 | — |

### 13.2 在插件中使用

```python
class MyPlugin(BasePlugin):
    async def on_load(self):
        # 注册权限路径
        self.rbac.register_permission("my_plugin.admin")
        self.rbac.register_permission("my_plugin.use")

        # 为角色分配权限
        self.rbac.assign_permission("admin", "my_plugin.*")
        self.rbac.assign_permission("user", "my_plugin.use")

    async def handle_message(self, event):
        msg = event.data
        # 检查权限
        if self.rbac.check_permission(msg.user_id, "my_plugin.admin"):
            await msg.reply(text="管理员功能")
        elif self.rbac.check_permission(msg.user_id, "my_plugin.use"):
            await msg.reply(text="普通功能")
        else:
            await msg.reply(text="无权限")
```

### 13.3 权限匹配树 (PermissionTrie)

内部使用前缀树（Trie）结构实现高效的权限匹配：

```
权限树示例：
root
├── chat
│   ├── group
│   │   ├── send     ← chat.group.send
│   │   ├── delete   ← chat.group.delete
│   │   └── *        ← chat.group.* (通配符)
│   └── private
│       └── send     ← chat.private.send
└── admin
    ├── kick         ← admin.kick
    └── ban          ← admin.ban
```

## 14. 完整插件示例

```python
from ncatbot.plugin_system import BasePlugin
from ncatbot.core.client import NcatBotEvent


class GreetPlugin(BasePlugin):
    """问候插件 — 根据时间段回复不同的问候语"""

    name = "greet_plugin"
    version = "1.0.0"
    author = "开发者"
    description = "根据时间段自动回复问候语"

    def _init_(self):
        """同步初始化"""
        self.greet_count = self.data.get("greet_count", 0)

    async def on_load(self):
        """异步初始化 — 注册事件和配置"""
        # 注册配置
        self.register_config("enabled", default=True, description="是否启用")
        self.register_config("prefix", default="你好", description="触发前缀")

        # 注册事件处理器
        self.register_handler(
            "ncatbot.message.group",
            self.handle_greet,
            priority=0
        )

        # 注册定时任务
        self.add_time_task("stats", self.log_stats, interval=3600)

    async def handle_greet(self, event: NcatBotEvent):
        """处理群消息"""
        msg = event.data
        prefix = self.get_config("prefix")

        if not self.get_config("enabled"):
            return

        if msg.raw_message.startswith(prefix):
            import time
            hour = time.localtime().tm_hour

            if 6 <= hour < 12:
                greeting = "早上好！"
            elif 12 <= hour < 18:
                greeting = "下午好！"
            else:
                greeting = "晚上好！"

            self.greet_count += 1
            self.data["greet_count"] = self.greet_count

            await msg.reply(text=f"{greeting}（第 {self.greet_count} 次问候）")

    async def log_stats(self):
        """定时输出统计"""
        print(f"已问候 {self.greet_count} 次")

    async def on_close(self):
        """清理 — 保存数据"""
        self.data["greet_count"] = self.greet_count
```
