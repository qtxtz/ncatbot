# 项目概览

## 1. 项目简介

NcatBot 是一款基于 **OneBot v11 协议** 的 Python QQ 机器人 SDK。它封装了与 [NapCat](https://github.com/NapNeko/NapCatQQ) 底层服务的通信细节，为开发者提供简洁的 Python 接口来开发 QQ 机器人应用。

### 核心特性

- **开箱即用**：自动管理 NapCat 的安装、配置和启动
- **事件驱动**：基于 asyncio 的异步事件系统，支持精确匹配、前缀匹配和正则匹配
- **插件化架构**：完整的插件生态系统，包括依赖管理、配置持久化、RBAC 权限控制
- **多平台支持**：Windows、Linux、macOS 全平台兼容
- **丰富 API**：100+ 个 API 方法，覆盖账号管理、群操作、消息发送等
- **CLI 工具**：命令行工具支持插件创建、Bot 启动、版本管理
- **MCP 支持**：集成 Model Context Protocol，实现 AI 辅助交互

## 4. 设计理念

### 4.1 五层架构模型

NcatBot 采用清晰的分层架构，每一层职责明确，层间通过接口通信：

```
┌─────────────────────────────────────────────────┐
│          第一层：客户端层（Client）               │
│    BotClient · EventBus · LifecycleManager      │
│    职责：统一入口、事件总线、生命周期管理          │
├─────────────────────────────────────────────────┤
│          第二层：事件层（Event）                  │
│    BaseEvent · EventParser · EventDispatcher    │
│    职责：事件定义、解析、分发                     │
├─────────────────────────────────────────────────┤
│          第三层：API 层（API）                    │
│    BotAPI · AccountAPI · GroupAPI · MessageAPI   │
│    职责：OneBot API 封装、请求-响应管理           │
├─────────────────────────────────────────────────┤
│          第四层：适配器层（Adapter）               │
│    NapCatService · WebSocket · AuthHandler       │
│    职责：NapCat 连接、认证、平台适配              │
├─────────────────────────────────────────────────┤
│          第五层：服务层（Service）                 │
│    ServiceManager · MessageRouter · RBAC        │
│    职责：可插拔服务管理、消息路由、权限控制        │
└─────────────────────────────────────────────────┘
```

### 4.2 核心设计原则

#### 事件驱动

整个框架以**事件**为核心运转。WebSocket 接收到的原始数据经过解析后成为事件对象，通过 EventBus 分发给所有匹配的处理器。开发者通过装饰器注册事件处理器，框架负责事件的路由和调度。

#### 依赖注入

服务、API、EventBus 等核心组件通过依赖注入提供给插件和事件处理器。插件不需要自行创建这些对象，框架会在加载阶段自动注入。

#### 单一职责

每个模块有明确的职责边界：
- `EventBus` 只负责事件的订阅和发布
- `EventDispatcher` 只负责将原始数据转换为事件
- `ServiceManager` 只负责服务的生命周期管理
- `PluginLoader` 只负责插件的发现和加载

#### 可扩展性

通过服务层和插件系统实现可扩展性：
- **服务层**：开发者可以注册自定义服务，与框架生命周期集成
- **插件系统**：完整的插件生态，支持依赖管理和版本控制
- **事件系统**：支持自定义事件类型和处理器

### 4.3 关键设计模式

| 设计模式 | 应用场景 | 实现说明 |
|----------|---------|---------|
| **单例模式** | BotClient | 通过 `_initialized` 类变量保证全局唯一 |
| **观察者模式** | EventBus | 发布-订阅机制，支持优先级和超时 |
| **工厂模式** | EventParser | 根据原始数据类型创建对应的事件对象 |
| **适配器模式** | IAPIClient | 统一不同通信方式的 API 调用接口 |
| **模板方法** | BaseService / BasePlugin | 定义生命周期骨架，子类实现具体逻辑 |
| **依赖注入** | ServiceManager → Plugin | 框架自动为插件注入所需的服务和 API |
| **组合模式** | BotAPI 多重继承 | 将多个 API 模块组合成统一接口 |
| **Future 模式** | MessageRouter | 使用 asyncio.Future + echo UUID 实现异步 RPC |
| **装饰器模式** | @bot.on_group_message() | 通过装饰器注册事件处理器 |
| **责任链模式** | 事件处理器链 | 按优先级排序执行，支持传播停止 |

## 6. 最简使用示例

```python
from ncatbot.core import BotClient, GroupMessage, PrivateMessage
from ncatbot.utils import ncatbot_config, get_log

_log = get_log()

# 配置
ncatbot_config.set_bot_uin("123456")       # QQ 号
ncatbot_config.set_root("123456")          # 管理员 QQ
ncatbot_config.set_ws_uri("ws://localhost:3001")

# 创建客户端（单例）
bot = BotClient()

# 注册群消息处理器
@bot.on_group_message()
async def on_group_message(msg: GroupMessage):
    if msg.raw_message == "测试":
        await msg.reply(text="NcatBot 测试成功喵~")

# 注册私聊消息处理器
@bot.on_private_message()
async def on_private_message(msg: PrivateMessage):
    if msg.raw_message == "测试":
        await msg.reply(text="NcatBot 测试成功喵~")

if __name__ == "__main__":
    bot.run()
```
