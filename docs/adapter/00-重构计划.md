# Adapter 层重构计划

## 1. 现状问题

### 1.1 结构混乱

当前 `ncatbot_legacy/adapter/napcat/` 的文件组织是扁平式的，所有组件平铺在同一层级：

```
ncatbot_legacy/adapter/napcat/
├── adapter.py          # 主入口，组合所有组件
├── api_impl.py         # IBotAPI 实现（散文式 20+ 方法）
├── api_impl/           # 但又有一个同名文件夹（？）
├── auth.py             # 登录认证（WebUI 交互）
├── config.py           # NapCat JSON 配置读写
├── connection.py       # WebSocket 连接管理
├── environment.py      # 安装/启动/登录流程编排
├── event_converter.py  # 事件转换（薄包装）
├── event_parser.py     # 实际事件解析逻辑
├── nc_service.py       # 与 environment.py 高度重复
├── nc_utils.py         # 下载/解压工具
├── platform.py         # Windows/Linux 平台差异
├── preupload/          # 预上传（唯一的子文件夹）
└── protocol.py         # echo/Future 请求-响应匹配
```

**具体问题**:

1. **职责不清晰**: `environment.py` 和 `nc_service.py` 逻辑高度重复（都做 WebSocket 测试、登录处理）；`config.py` 与 `platform.py`/`auth.py` 都涉及配置和环境管理
2. **setup 流程散落多处**: 安装→配置→启动→登录的流程分散在 `environment.py`、`nc_service.py`、`platform.py`、`config.py`、`auth.py` 五个文件中，依赖关系不明确
3. **connection 和 protocol 耦合**: `connection.py` 只做 WebSocket I/O，`protocol.py` 做 echo 匹配——但事件回调和重连逻辑互相交叉引用
4. **event_converter 是空壳**: `event_converter.py` 只是 `EventParser.parse()` 的一层包装，没有实际转换逻辑
5. **preupload 独立成包但不完整**: 预上传是唯一分了子目录的模块，但 `service.py` 里的逻辑本应和 `api_impl.py` 更紧密
6. **api_impl 过于冗长**: 20+ 方法全部写在一个文件里，每个方法都是 `_call("action", {params})` 的重复模式
7. **旧引用残留**: `connection.py` 的重连逻辑直接 import 了 `ncatbot.core.adapter.nc.service`（旧路径），`environment.py` 也引用了旧的 `ncatbot.core.adapter.nc.*`

### 1.2 BaseAdapter 接口设计问题

- `convert_event()` 方法不应由 Adapter 公开暴露——事件转换是内部实现细节
- `set_event_callback()` + `_event_callback` 是一个脆弱的回调管道，应直接对接新的异步分发器
- `preupload_file()` / `preupload_message()` 放在 BaseAdapter 上作为可选方法，但实际逻辑很复杂，不应该是简单的方法覆盖

### 1.3 EventBus/BusEvent 设计问题

旧的 `EventBus` + `BusEvent`/`NcatBotEvent` 存在以下问题：
- `BusEvent.data` 携带的是 `BaseEvent` 实体（持有 API 引用），与纯数据模型理念冲突
- 实体层（`event/` 下的 `BaseEvent`、`MessageEvent` 等）将数据和行为捆绑在一起，但 adapter 层不应关心行为——行为应该由分发器/插件层按需注入
- `EventBus` 是一个通用的 pub/sub 引擎，但事件分发的实际需求是类型化的异步分发（按 `post_type` + 子类型路由），不需要全功能 pub/sub
- `create_entity()` 在 adapter 层创建实体，导致 adapter 同时依赖 `types/` 和 `event/`；实际上 adapter 只需依赖 `types/`

### 1.4 与新 types 层脱节

新的 `ncatbot/types/` 数据模型层已完成：
- `types/` = 纯 Pydantic 数据模型（零依赖，可序列化）
- `EventParser.parse(raw_dict)` → `BaseEventData`

但旧 adapter 仍然引用 `ncatbot.core.event.parser.EventParser`（旧版），完全没有对接新的 types 层。

## 2. 新设计

### 2.1 设计原则

1. **按职责域分文件夹** — setup、connection、service 各自独立
2. **adapter 主类只做编排** — 组合子组件，不包含实际逻辑
3. **只产出数据模型** — adapter 只依赖 `types/`，产出 `BaseEventData`（纯 Pydantic 模型），不创建实体
4. **异步分发器替代 EventBus** — 新分发器直接携带数据模型，由分发器层按需注入 API 引用和行为
5. **消除重复** — nc_service/environment 合并，event_converter/event_parser 用新 types 层替代
6. **IBotAPI 按分类拆分** — 消息 / 群管 / 查询 / 文件 分文件
7. **BaseAdapter 接口简化** — 去掉 `convert_event`，回调签名改为 `Callable[[BaseEventData], Awaitable[None]]`

### 2.3 BaseAdapter 精简设计

```python
# ncatbot/adapter/base.py
from abc import ABC, abstractmethod
from typing import Optional, Callable, Awaitable, List, TYPE_CHECKING

if TYPE_CHECKING:
    from ncatbot.types import BaseEventData

class BaseAdapter(ABC):
    """平台适配器抽象基类

    回调签名为 Callable[[BaseEventData], Awaitable[None]]，
    即 adapter 只产出纯数据模型，不创建实体。
    """

    name: str
    description: str
    supported_protocols: List[str]

    _event_callback: Optional[Callable[["BaseEventData"], Awaitable[None]]] = None

    # ---- 生命周期 ----

    @abstractmethod
    async def setup(self) -> None:
        """准备平台环境（安装/配置/启动）"""

    @abstractmethod
    async def connect(self) -> None:
        """建立连接并初始化 API"""

    @abstractmethod
    async def disconnect(self) -> None:
        """断开连接，释放资源"""

    @abstractmethod
    async def listen(self) -> None:
        """阻塞监听消息，内部完成事件解析后回调数据模型"""

    # ---- API ----

    @abstractmethod
    def get_api(self) -> "IBotAPI":
        """返回 IBotAPI 实现"""

    # ---- 回调 ----

    def set_event_callback(
        self, callback: Callable[["BaseEventData"], Awaitable[None]]
    ) -> None:
        """设置事件数据回调，由异步分发器在启动时调用"""
        self._event_callback = callback

    # ---- 状态 ----

    @property
    @abstractmethod
    def connected(self) -> bool: ...
```

**变更要点**:
- 移除 `convert_event()` — 事件转换是内部实现细节，不属于公共接口
- **回调签名改为 `Callable[[BaseEventData], Awaitable[None]]`** — adapter 只产出数据模型，不产出实体
- 移除 `preupload_file()` / `preupload_message()` — 预上传是 NapCat 特有能力，不属于通用接口。需要时由 `NapCatBotAPI` 内部调用。

### 2.4 各模块职责明细

#### setup/ — 环境准备

| 文件 | 职责 | 输入 | 输出 |

**launcher.py** 取代旧的 `environment.py` + `nc_service.py`：

```python
class NapCatLauncher:
    """NapCat 启动编排"""
    def __init__(self):
        self._platform = PlatformOps.create()
        self._installer = NapCatInstaller(self._platform)
        self._config = NapCatConfig(self._platform)
        self._auth = AuthHandler()

    async def launch(self) -> None:
        """执行完整的启动流程"""
        if ncatbot_config.napcat.remote_mode:
            await self._launch_remote()
        else:
            await self._launch_local()

    async def _launch_local(self) -> None:
        self._installer.ensure_installed()   # 安装/更新
        self._config.configure_all()         # 写配置
        self._platform.start_napcat(uin)     # 启动进程
        await self._auth.ensure_logged_in()  # 登录
```

#### connection/ — 网络连接

| 文件 | 职责 |
|------|------|
| `websocket.py` | WebSocket 建连/断连/重连/消息收发 |
| `protocol.py` | echo UUID → Future 匹配；区分 API 响应和事件推送 |

**关键改进**:
- `websocket.py` 不再 import 任何 service/adapter 模块（重连不应该依赖 NapCatService）
- 重连策略作为参数注入或使用简单的指数退避
- `protocol.py` 只做匹配，不处理事件回调（事件回调由 adapter 层统一处理）

```python
class NapCatWebSocket:
    """纯 WebSocket 连接管理"""
    async def connect(self, uri: str) -> None: ...
    async def disconnect(self) -> None: ...
    async def send(self, data: dict) -> None: ...
    async def listen(self, on_message: Callable) -> None: ...
    # 重连策略
    async def _reconnect(self) -> bool:
        """指数退避重连，不依赖外部模块"""

class OB11Protocol:
    """OneBot11 请求-响应协议"""
    async def call(self, action: str, params: dict, timeout: float = 30) -> dict: ...
    def set_event_handler(self, handler: Callable): ...
    async def on_message(self, data: dict) -> None:  # 由 websocket.listen 调用
```

#### parser.py — 事件解析胶水

```python
# ncatbot/adapter/napcat/parser.py
from ncatbot.event import EventParser
from ncatbot.types import BaseEventData

class NapCatEventParser:
    """将 OB11 原始 JSON 转换为 BaseEventData 数据模型

    只产出纯数据模型，不创建实体。
    实体创建（如需要）由分发器层按需完成。
    """

    def parse(self, raw_data: dict) -> Optional[BaseEventData]:
        if "post_type" not in raw_data:
            return None
        return EventParser.parse(raw_data)  # dict → types/ 数据模型，到此为止
```

**关键变化**: `NapCatEventParser` 不再需要 `api` 参数，因为不创建实体。parser 的输出是纯数据模型，可序列化、可独立测试。

### 2.5 NapCatAdapter 主类

```python
# ncatbot/adapter/napcat/adapter.py
class NapCatAdapter(BaseAdapter):
    name = "napcat"
    description = "NapCat (OneBot v11) 适配器"
    supported_protocols = ["onebot_v11"]

    def __init__(self):
        self._launcher = NapCatLauncher()
        self._ws: Optional[NapCatWebSocket] = None
        self._protocol: Optional[OB11Protocol] = None
        self._api: Optional[NapCatBotAPI] = None
        self._parser = NapCatEventParser()  # 无需 api 参数

    async def setup(self) -> None:
        await self._launcher.launch()

    async def connect(self) -> None:
        uri = ncatbot_config.get_uri_with_token()
        self._ws = NapCatWebSocket()
        await self._ws.connect(uri)
        self._protocol = OB11Protocol(self._ws)
        self._api = NapCatBotAPI(self._protocol)
        self._protocol.set_event_handler(self._on_event)

    async def disconnect(self) -> None:
        if self._protocol:
            self._protocol.cancel_all()
        if self._ws:
            await self._ws.disconnect()
        self._api = self._protocol = self._ws = None

    async def listen(self) -> None:
        await self._ws.listen(self._protocol.on_message)

    def get_api(self) -> IBotAPI:
        return self._api

    @property
    def connected(self) -> bool:
        return self._ws is not None and self._ws.connected

    async def _on_event(self, raw_data: dict) -> None:
        """收到事件推送，解析为数据模型后回调给分发器"""
        data_model = self._parser.parse(raw_data)
        if data_model and self._event_callback:
            await self._event_callback(data_model)  # 传递 BaseEventData
```

### 2.6 数据流总览

```
NapCat WebSocket 原始 JSON
  ↓
NapCatWebSocket.listen() → on_message(data)
  ↓
OB11Protocol.on_message(data)
  ├─ 有 echo → resolve Future → return API 响应给调用方
  └─ 无 echo → event_handler(data)
      ↓
NapCatAdapter._on_event(data)
  ↓
NapCatEventParser.parse(data)
  └─ EventParser.parse(data)   → BaseEventData (types/) ← 到此为止，不创建实体
      ↓
_event_callback(data_model)    → 传递纯数据模型
  ↓
AsyncEventDispatcher.dispatch(data_model, api)
  ├─ 按 post_type + 子类型路由到 handler
  ├─ handler 收到 (data_model, api) 或包装后的上下文对象
  │     ├─ data_model.user_id   ← 直接访问字段
  │     ├─ data_model.message.text
  │     ├─ api.send_group_msg(...)  ← 通过 api 参数调用
  │     └─ data_model.model_dump()  ← 纯数据序列化
  └─ 支持 stop_propagation / 优先级 / 超时
```

### 2.7 异步分发器设计（替代 EventBus + BusEvent）

旧的 `EventBus` + `BusEvent` / `NcatBotEvent` 被一个轻量级异步分发器替代：

```python
# ncatbot/core/dispatcher/dispatcher.py
from ncatbot.types import BaseEventData
from ncatbot.api.interface import IBotAPI

class AsyncEventDispatcher:
    """异步事件分发器

    只携带数据模型，不创建实体。
    handler 签名: async def handler(data: BaseEventData, api: IBotAPI) -> None
    """

    def __init__(self, api: IBotAPI):
        self._api = api
        self._handlers: Dict[str, List[HandlerEntry]] = {}  # event_type → handlers

    def subscribe(
        self,
        event_type: str,          # e.g. "message.group", "notice.group_increase"
        handler: EventHandler,
        priority: int = 0,
    ) -> str:
        """注册事件处理器"""
        ...

    async def dispatch(self, data: BaseEventData) -> None:
        """分发数据模型到匹配的 handler"""
        event_type = self._resolve_type(data)  # 从 data.post_type + 子类型推导
        handlers = self._collect_handlers(event_type)
        for entry in sorted(handlers, key=lambda h: -h.priority):
            try:
                result = await asyncio.wait_for(
                    entry.handler(data, self._api),
                    timeout=entry.timeout,
                )
                if result is STOP_PROPAGATION:
                    break
            except asyncio.TimeoutError:
                LOG.warning(f"Handler {entry.name} 超时")
            except Exception as e:
                LOG.error(f"Handler {entry.name} 异常: {e}")

    def _resolve_type(self, data: BaseEventData) -> str:
        """从数据模型推导事件类型字符串

        例:
          MessageEventData(message_type=GROUP) → "message.group"
          GroupIncreaseNoticeEventData → "notice.group_increase"
          FriendRequestEventData → "request.friend"
          HeartbeatMetaEventData → "meta.heartbeat"
        """
        ...
```

**与旧 EventBus 的对比**:

| 对比项 | 旧 EventBus + BusEvent | 新 AsyncEventDispatcher |
|--------|----------------------|-------------------------|
| 携带数据 | `BaseEvent` 实体（持有 API） | `BaseEventData` 纯数据模型 |
| API 注入 | 实体构造时绑定 | 分发时作为参数传递 |
| 匹配模式 | 精确/前缀/正则 | 类型化路由（按 post_type + 子类型） |
| 停止传播 | `event.stop_propagation()` | handler 返回 `STOP_PROPAGATION` 哨兵值 |
| 线程安全 | `publish_threadsafe()` | 不需要（纯异步） |
| 结果收集 | `NcatBotEvent._results` | 不收集（handler 自行处理） |

```python
    async def get_msg(self, message_id) -> dict: ...
    @abstractmethod
    async def get_forward_msg(self, id) -> dict: ...

    # ---- 文件操作 ----
    @abstractmethod
    async def upload_group_file(self, group_id, file, name, folder="") -> None: ...
    @abstractmethod
    async def get_group_root_files(self, group_id) -> dict: ...
    @abstractmethod
    async def get_group_file_url(self, group_id, file_id, busid) -> dict: ...
    @abstractmethod
    async def delete_group_file(self, group_id, file_id, busid) -> None: ...

    # ---- 其他 ----
    @abstractmethod
    async def send_like(self, user_id, times=1) -> None: ...
    @abstractmethod
    async def send_poke(self, user_id, group_id=None) -> None: ...
```

**单向依赖**:
- `adapter/napcat/` → `types/` (数据模型) — **不再依赖 `event/`**
- `adapter/napcat/` → `api/interface` (IBotAPI 抽象)
- `core/dispatcher/` → `types/` (数据模型) + `api/interface` (IBotAPI)
- `adapter/napcat/` 内部: setup/ ← connection/ ← api/ ← parser.py ← adapter.py
