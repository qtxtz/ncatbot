## 1. 架构总览

消息体系的核心是三层嵌套结构：

```
MessageEvent
├── message: MessageArray              # 一条消息由多个消息段组成
│   ├── message: List[MessageSegment]  # 消息段列表
│   │   ├── PlainText(text="hello ")   # 文本段
│   │   ├── At(qq="123456")            # @某人段
│   │   └── Image(file="test.jpg")     # 图片段
│   │   └── ...
├── raw_message: str                   # CQ 码原始文本
├── message_id: str                    # 消息 ID
├── user_id: str                       # 发送者 ID
├── sender: BaseSender / GroupSender   # 发送者信息
└── (group_id: str)                    # 群聊特有
```

**设计依赖关系：**

```
MessageSegment  ← 被 MessageArray 持有（1:N）
       ↑
MessageArray    ← 被 MessageEvent.message 持有（1:1）
       ↑
MessageEvent    ← 由 EventDispatcher 创建并分发
```

**协议基础：** 整个消息体系基于 [OneBot v11 协议](https://github.com/botuniverse/onebot-11) 的消息格式规范，消息以段数组（Segment Array）的形式表示。

## 2. MessageSegment 消息段

> 源文件：`ncatbot/core/event/message_segments/base.py`

### 2.1 基类设计

```python
class MessageSegment(BaseModel, ABC):
    model_config = ConfigDict(extra="allow", arbitrary_types_allowed=True)
    type: ClassVar[str]  # 每个子类用 ClassVar 定义自己的类型标识
```

关键设计点：
- 继承 Pydantic `BaseModel` + `ABC`（抽象基类），兼具数据校验和抽象约束能力
- `type` 使用 `ClassVar[str]`，不参与实例序列化（`model_dump` 时 `exclude={"type"}`）
- `extra="allow"`：允许接收协议中未定义的额外字段，避免因 OneBot 实现差异导致反序列化失败
- `arbitrary_types_allowed=True`：允许非标准类型字段

### 2.2 TYPE_MAP 自动注册机制

```python
TYPE_MAP: Dict[str, Type["MessageSegment"]] = {}

class MessageSegment(BaseModel, ABC):
    def __init_subclass__(cls, **kwargs: Any) -> None:
        super().__init_subclass__(**kwargs)
        if hasattr(cls, "type") and isinstance(getattr(cls, "type", None), str):
            TYPE_MAP[cls.type] = cls
```

**工作原理：**
- 所有 `MessageSegment` 的子类在**类定义时**自动注册到 `TYPE_MAP`
- 条件 `isinstance(getattr(cls, "type", None), str)` 确保只有定义了具体 `type` 字符串值的子类才注册，跳过中间抽象类（如 `DownloadableMessageSegment`）
- 注册后的 `TYPE_MAP` 用于 `parse_message_segment()` 的类型路由

**当前注册表：**

| type 值 | 类名 | 所在文件 |
|---------|------|----------|
| `"text"` | `PlainText` | `primitives.py` |
| `"face"` | `Face` | `primitives.py` |
| `"at"` | `At` | `primitives.py` |
| `"reply"` | `Reply` | `primitives.py` |
| `"image"` | `Image` | `media.py` |
| `"record"` | `Record` | `media.py` |
| `"video"` | `Video` | `media.py` |
| `"file"` | `File` | `media.py` |
| `"share"` | `Share` | `misc.py` |
| `"location"` | `Location` | `misc.py` |
| `"music"` | `Music` | `misc.py` |
| `"json"` | `Json` | `misc.py` |
| `"markdown"` | `Markdown` | `misc.py` |
| `"forward"` | `Forward` | `forward.py` |

### 2.3 序列化与反序列化

**OneBot v11 消息段格式：**

```json
{
    "type": "text",
    "data": {
        "text": "hello"
    }
}
```

**反序列化（dict → MessageSegment）：**

```python
# 入口：parse_message_segment()
def parse_message_segment(data: Dict[str, Any]) -> MessageSegment:
    seg_type = data.get("type")          # 取 "type" 字段
    target_cls = TYPE_MAP.get(seg_type)  # 查 TYPE_MAP 路由到具体类
    return target_cls.from_dict(data)    # 调用该类的 from_dict

# 基类默认 from_dict 实现：
@classmethod
def from_dict(cls, data: Dict[str, Any]) -> "MessageSegment":
    obj = cls(**data.get("data", {}))  # 取 "data" 子字典，展开为构造参数
    return obj
```

**序列化（MessageSegment → dict）：**

```python
def to_dict(self) -> Dict[str, Any]:
    dump = self.model_dump(exclude={"type"}, exclude_none=True)
    return {"type": self.type, "data": dump}
```

**注意：** `model_dump(exclude={"type"})` 排除 `type` 字段是因为 `type` 是 `ClassVar`，不包含在 Pydantic 的 `model_fields` 中，但 `exclude` 仍然指定以确保安全性。`exclude_none=True` 略去值为 `None` 的可选字段。

#### 2.4.4 转发类型 (`forward.py`)

**`Node` 模型（非 MessageSegment 子类）：**

```python
class Node(BaseModel):
    user_id: str                # 发送者 ID
    nickname: str               # 发送者昵称
    content: MessageArrayDTO    # 该节点的消息内容
```

**`Forward` 消息段：**

```python
class Forward(MessageSegment):
    type: ClassVar[str] = "forward"
    id: Optional[str] = None           # 引用已有转发消息的 ID
    content: Optional[List[Node]] = None  # 自建转发的完整内容
```

Forward 有两种模式：
1. **引用模式**：只有 `id`，引用一条已存在的转发消息
2. **内容模式**：有 `content`，包含完整的 `Node` 列表，支持嵌套转发

### 2.5 特殊序列化逻辑

部分消息段类型重写了 `from_dict` / `to_dict`，因为 OneBot 协议的 `data` 字典键名与 Python 类字段名存在冲突：

#### Image 的 `type` 字段冲突

```python
# OneBot 协议 Image 格式：
{"type": "image", "data": {"file": "test.jpg", "type": 1}}
#                                                ^^^^
# data.type 与外层 type 键名冲突

# 解决方案：from_dict 时将 data["type"] 映射为 image_type 字段
@classmethod
def from_dict(cls, data):
    if "type" in data.get("data", {}):
        data["data"]["image_type"] = data["data"].pop("type")
    return super().from_dict(data)

# to_dict 时反向转换
def to_dict(self):
    data = super().to_dict()
    if "image_type" in data["data"] and data["data"]["image_type"] is not None:
        image_type = data["data"].pop("image_type")
        data["data"]["type"] = 1 if image_type == "flash" else 0
    return data
```

#### Music 的 `type` 字段冲突

```python
# OneBot 协议 Music 格式：
{"type": "music", "data": {"type": "qq", "id": "123"}}
#                           ^^^^
# data.type 表示音乐平台，与外层 type 冲突

# 解决方案：from_dict 时映射为 platform 字段
@classmethod
def from_dict(cls, data):
    data["data"]["platform"] = data["data"]["type"]
    del data["data"]["type"]
    return super().from_dict(data)

# to_dict 时反向转换
def to_dict(self):
    data = super().to_dict()
    data["data"]["type"] = data["data"]["platform"]
    del data["data"]["platform"]
    return data
```

#### Forward 的复杂序列化

Forward 完全重写了 `from_dict` 和 `to_dict`：

- **`to_dict()`**：根据有 `content` 还是只有 `id` 输出不同格式
- **`from_dict()`**：需要处理嵌套的 message 事件格式（含 `"message"` 键的 dict → 转换为 `Node`）
- **`to_forward_dict()`**：生成 `send_forward_msg` API 调用所需的 `{"messages": [...]}` 格式
- **`Node.to_node_dict()`**：处理嵌套转发——如果 Node.content 中包含 Forward 类型段，将其展开为节点列表（递归扁平化）

## 3. MessageArray 消息数组

> 源文件：`ncatbot/core/event/message_segments/message_array.py`

### 3.1 继承链 MessageArrayDTO → MessageArray

```
BaseModel
  └── MessageArrayDTO         # 数据传输对象，只持有 message 列表及基本序列化方法
        └── MessageArray      # 添加了构建器方法、过滤方法、运算符重载
```

**MessageArrayDTO（DTO 基类）：**

```python
class MessageArrayDTO(BaseModel):
    model_config = ConfigDict(extra="allow", arbitrary_types_allowed=True)
    message: List[MessageSegment]  # 消息段列表

    @classmethod
    def from_list(cls, data: List[Dict]) -> "MessageArrayDTO":
        return cls(message=[parse_message_segment(seg) for seg in data])

    def to_list(self) -> List[Dict]:
        return [seg.to_dict() for seg in self.message]
```

- 作为纯数据容器，Forward 模块中的 `Node.content` 使用 `MessageArrayDTO` 类型（而非 MessageArray），避免引入 MessageArray 的复杂依赖
- `from_list` / `to_list` 完成与 OneBot v11 JSON 数组的互转

**MessageArray（完整封装）：**

继承 MessageArrayDTO，增加了构建（Builder）、过滤（Filter）、运算符、兼容性等能力。

### 3.2 构造方式

MessageArray 有多种构造方式：

```python
# 1. 空构造
msg = MessageArray()

# 2. 从 MessageSegment 对象
msg = MessageArray(PlainText(text="hello"))

# 3. 从字符串（自动解析 CQ 码）
msg = MessageArray("hello [CQ:at,qq=123]")

# 4. 从多个参数（混合类型）
msg = MessageArray("text", At(qq="123"), Image(file="test.jpg"))

# 5. 从列表（内部调用 from_list）
msg = MessageArray.from_list([
    {"type": "text", "data": {"text": "hello"}},
    {"type": "at", "data": {"qq": "123"}}
])

# 6. 通过 message 关键字参数（直接传段列表）
msg = MessageArray(message=[PlainText(text="hello")])
```

**构造函数逻辑：**

```python
def __init__(self, *args, **kwargs):
    if "message" in kwargs:
        super().__init__(**kwargs)         # 直接传 message 列表
    else:
        super().__init__(message=parse_message_segments(args))  # 解析可变参数
```

### 3.3 构建器方法（Builder Pattern）

所有构建器方法均返回 `self` 以支持链式调用：

| 方法 | 参数 | 说明 |
|------|------|------|
| `add_text(text: str)` | 文本字符串 | 添加文本段，**自动解析 CQ 码** |
| `add_at(user_id)` | QQ号 | 添加 @某人 |
| `add_at_all()` | 无 | 添加 @全体成员 |
| `add_reply(message_id)` | 消息 ID | 添加回复引用 |
| `add_image(image)` | 路径/URL/Image 对象 | 添加图片 |
| `add_by_segment(segment)` | MessageSegment | 添加任意消息段 |
| `add_by_dict(data)` | OneBot 格式 dict | 从字典添加 |
| `add_by_list(data)` | dict 或 MessageSegment 列表 | 批量添加 |

**链式调用示例：**

```python
msg = (
    MessageArray()
    .add_reply("12345")
    .add_at("67890")
    .add_text(" 你好！")
    .add_image("http://example.com/pic.jpg")
)
```

**注意 `add_text` 的行为：** 它调用 `parse_message_segments(text)`，会解析字符串中的 CQ 码。这意味着 `add_text("[CQ:at,qq=123]hello")` 会产生一个 `At` 段和一个 `PlainText` 段，而不是一个包含 CQ 码原文的纯文本段。

### 3.4 解析与过滤方法

| 方法 | 返回类型 | 说明 |
|------|----------|------|
| `concatenate_text()` | `str` | 拼接所有 PlainText 段的文本 |
| `filter(cls=None)` | `List[MessageSegment]` 或 `List[T]` | 按类型过滤消息段 |
| `filter_text()` | `List[PlainText]` | 过滤文本段 |
| `filter_at()` | `List[At]` | 过滤 @段 |
| `filter_image()` | `List[Image]` | 过滤图片段 |
| `filter_video()` | `List[Video]` | 过滤视频段 |
| `filter_face()` | `List[Face]` | 过滤表情段 |
| `is_user_at(user_id, all_except=False)` | `bool` | 检查消息是否 @了某用户 |
| `is_forward_msg()` | `int`（实际表示 bool） | 检查是否包含转发消息 |

**`is_user_at` 逻辑说明：**

```python
def is_user_at(self, user_id, all_except=False):
    # 遍历所有 At 段
    # 1. 如果某个 At 的 qq == user_id → 返回 True（精确 @）
    # 2. 如果某个 At 的 qq == "all"（@全体）：
    #    - all_except=False → 返回 True（@全体也算 @了某用户）
    #    - all_except=True  → 不算（排除 @全体）
```

### 3.5 运算符重载

```python
msg1 = MessageArray("hello ")
msg2 = MessageArray(At(qq="123"))

# 拼接操作
combined = msg1 + msg2           # MessageArray + MessageArray
combined = msg1 + "world"        # MessageArray + str
combined = msg1 + At(qq="456")   # MessageArray + MessageSegment
combined = "prefix " + msg1      # str + MessageArray（__radd__）
```

### 3.6 CQ 码解析

> 函数 `parse_cq_code_to_onebot11(cq_string: str) -> List[Dict]`

将 CQ 码字符串解析为 OneBot v11 消息数组格式：

**输入：**
```
"hello [CQ:at,qq=123456]你好[CQ:image,file=test.jpg]"
```

**输出：**
```python
[
    {"type": "text", "data": {"text": "hello "}},
    {"type": "at", "data": {"qq": "123456"}},
    {"type": "text", "data": {"text": "你好"}},
    {"type": "image", "data": {"file": "test.jpg"}}
]
```

**HTML 实体转义处理：**

| 转义 | 原字符 |
|------|--------|
| `&amp;` | `&` |
| `&#91;` | `[` |
| `&#93;` | `]` |
| `&#44;` | `,` |

解析流程：
1. 用正则 `\[CQ:([^,\]]+)(?:,([^\]]+))?\]` 匹配所有 CQ 码
2. CQ 码之间和前后的普通文本作为 `text` 段
3. CQ 码拆解为 `type` + `params`（逗号分隔的 key=value 对）
4. 所有文本和参数值都做 HTML 实体反转义

### 3.7 通用解析函数 parse_message_segments

```python
def parse_message_segments(data: Any) -> List[MessageSegment]:
```

这是一个递归通用解析器，处理所有可能的输入类型：

| 输入类型 | 处理方式 |
|----------|----------|
| `dict` | 调用 `parse_message_segment(data)` 解析单个段 |
| `MessageSegment` 实例 | 直接包装为列表返回 |
| `str` | 调用 `parse_cq_code_to_onebot11` 解析 CQ 码后再逐段解析 |
| `AtAll` 实例 | 转换为 `At(qq="all")` |
| `Iterable`（列表/元组等） | 递归处理每个元素 |
| 其他 | 返回空列表 |

## 4. MessageEvent 消息事件

> 源文件：`ncatbot/core/event/events.py`

### 4.1 继承层次

```
BaseModel (Pydantic)
  └── ContextMixin                        # API 依赖注入（_api 属性）
        └── BaseEvent                     # time, self_id, post_type
              └── MessageEvent            # + MessageActionMixin
                    ├── PrivateMessageEvent   # 私聊
                    └── GroupMessageEvent     # 群聊，+ GroupAdminMixin
```

**Mixin 组合：**

```
MessageEvent = BaseEvent + MessageActionMixin
GroupMessageEvent = MessageEvent + GroupAdminMixin
```

### 4.2 字段定义

#### BaseEvent

| 字段 | 类型 | 说明 |
|------|------|------|
| `time` | `int` | 事件时间戳（Unix 秒） |
| `self_id` | `str` | 机器人 QQ 号（自动 int→str） |
| `post_type` | `PostType` | 事件类型枚举 |

#### MessageEvent（在 BaseEvent 基础上）

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `post_type` | `PostType` | `PostType.MESSAGE` | 固定为 `"message"` |
| `message_type` | `MessageType` | — | `"private"` 或 `"group"` |
| `sub_type` | `str` | — | 私聊: `"friend"`; 群聊: `"normal"` / `"anonymous"` |
| `message_id` | `str` | — | 消息 ID（自动 int→str） |
| `user_id` | `str` | — | 发送者 QQ 号（自动 int→str） |
| `message` | `MessageArray` | — | **消息内容**（自动 list→MessageArray） |
| `raw_message` | `str` | — | CQ 码原始文本（由 OneBot 实现提供） |
| `sender` | `BaseSender` | — | 发送者信息 |
| `font` | `int` | `0` | 字体编号 |

#### PrivateMessageEvent（在 MessageEvent 基础上）

### 4.3 字段自动转换

MessageEvent 定义了多个 `@field_validator`，在 Pydantic 反序列化时自动做类型转换：

```python
# 1. self_id：int → str（在 BaseEvent 中）
@field_validator("self_id", mode="before")
def force_str(cls, v):
    return str(v)

# 2. message_id, user_id：int → str
@field_validator("message_id", "user_id", mode="before")
def _force_ids(cls, v):
    return str(v)

# 3. message：List[dict] → MessageArray
@field_validator("message", mode="before")
def _convert_message(cls, v):
    if isinstance(v, list):
        return MessageArray.from_list(v)
    return v

# 4. group_id：int → str（在 GroupMessageEvent 中）
@field_validator("group_id", mode="before")
def _gid(cls, v):
    return str(v)
```

这意味着原始 JSON 中的数值型 ID 会被统一为字符串类型，消息数组会被自动解析为 `MessageArray` 对象。

### 4.4 Mixin 能力注入

#### ContextMixin（API 上下文绑定）

> 源文件：`ncatbot/core/event/context.py`

```python
class ContextMixin(BaseModel):
    _api: Optional[IBotAPI] = PrivateAttr(default=None)

    def bind_api(self, api: IBotAPI):
        self._api = api

    @property
    def api(self) -> IBotAPI:
        if self._api is None:
            raise RuntimeError("API context not initialized.")
        return self._api
```

- 使用 Pydantic `PrivateAttr` 存储 API 引用，不参与序列化
- EventDispatcher 在创建事件后调用 `event.bind_api(api)` 注入 API 上下文
- 事件对象通过 `self.api` 即可调用 bot API

#### MessageActionMixin（消息操作）

## 7. 兼容层与历史遗留

### 7.1 Text 和 AtAll（4.4.x 兼容）

```python
# message_array.py 中定义了兼容类型
@dataclass
class Text(str):          # 兼容旧版 Text 类型
    type: ClassVar[str] = "text"
    text: str

@dataclass
class AtAll:              # 兼容旧版 AtAll 类型
    type: ClassVar[str] = "at"
    qq: str = "all"
```

- `Text` 继承 `str` 且是 `dataclass`，用于兼容 4.4.x 中将 `Text` 当字符串使用的写法
- `AtAll` 是独立于 `At` 的 dataclass，在 `parse_message_segments` 中被转为 `At(qq="all")`
- 这些类型**不注册** `TYPE_MAP`（因为不是 MessageSegment 子类）

### 7.2 MessageArray.messages 属性

```python
@property
def messages(self) -> List[MessageSegment]:
    return self.message
```

`messages`（复数）是 `message` 的只读别名，用于向后兼容旧代码中的 `msg.messages` 访问方式。

### 7.3 核心模块别名导出

```python
# ncatbot/core/__init__.py
from .event import GroupMessageEvent as GroupMessage
from .event import PrivateMessageEvent as PrivateMessage
from .event import MessageEvent as BaseMessageEvent
from .event import MessageEvent as MessageSentEvent
```

提供简写别名以兼容不同命名风格和旧 API。

## 9. 已知问题与重构建议

### 9.1 `message` 字段命名歧义

- `MessageEvent.message` 类型是 `MessageArray`，但名字是单数 `message`
- `MessageArrayDTO.message` 也是 `List[MessageSegment]`，同样是单数
- `MessageArray.messages` 是 `message` 的别名属性
- 造成在不同上下文中 `message` 可能指代：一条完整消息（MessageArray）、消息段列表（List[MessageSegment]）、或 API 参数（List[dict]）

### 9.2 `MessageActionMixin.reply` 的参数类型

```python
async def reply(self, text: str, **kwargs):
```

参数名是 `text`，类型标注是 `str`，但实际传给 `api.send_group_msg` 的 `message` 参数。这限制了 reply 只能发纯文本，无法直接传 `MessageArray`。

### 9.3 `is_forward_msg` 返回值类型

```python
def is_forward_msg(self):
    return len(self.filter(Forward))  # 返回 int，而非 bool
```

方法名暗示返回 `bool`，但实际返回 `int`（Forward 段的数量）。虽然 Python 中 truthy 语义兼容，但类型不一致。

### 9.4 Image / Music 的字段名冲突处理

当前通过重写 `from_dict` / `to_dict` 解决字段名冲突（`data.type` vs 外层 `type`），这使得序列化逻辑分散在各子类中，增加了维护成本。可考虑在基类层面提供字段映射机制。

### 9.5 CQ 码在 `add_text` 中的自动解析

`add_text` 会自动解析传入文本中的 CQ 码，这可能是非预期行为。如果开发者只想添加纯文本（即使其中包含 `[CQ:...]` 格式的字面量），目前没有简单方法绕过解析。

### 9.6 Text 兼容类型的混乱

`Text` 同时继承 `str` 和使用 `@dataclass` 装饰器，且有 `type: ClassVar[str] = "text"` 属性，但它不是 `MessageSegment` 子类。它与 `PlainText` 功能重叠，存在的唯一理由是 4.4.x 兼容。

### 9.7 Forward.from_dict 的容错逻辑

`Forward.from_dict` 中有大量容错代码处理不完整的 content 数据（如 `...` 省略、message 事件格式 → Node 格式转换）。这暗示上游数据格式不稳定或存在多种变体需要适配。

### 9.8 MessageArrayDTO 与 MessageArray 的职责边界

`MessageArrayDTO` 设计为纯数据传输对象，`MessageArray` 在其上添加了业务逻辑。但 `Forward` 的 `Node.content` 使用 `MessageArrayDTO` 类型，这意味着 Node 中的消息无法使用 `filter`、`concatenate_text` 等便捷方法。如果后续需要在 Node 层面操作消息内容，这会是一个限制。
