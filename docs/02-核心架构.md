# 核心架构

本文档详细介绍 NcatBot 的核心架构设计，包括 BotClient 的组成、生命周期管理、事件总线机制和启动流程。

> 源码位置：`ncatbot/core/client/`

## 1. BotClient — 统一入口

`BotClient` 是整个框架的入口类，采用**单例模式**确保全局唯一。它通过多重继承组合了 `EventRegistry`（事件注册）和 `LifecycleManager`（生命周期管理）两个功能模块。

### 1.1 类结构

```python
class BotClient(EventRegistry, LifecycleManager):
    """
    继承关系：
    - EventRegistry：提供 @bot.on_group_message() 等装饰器
    - LifecycleManager：提供 run()、shutdown() 等生命周期方法
    """
```

### 1.2 核心属性

| 属性 | 类型 | 说明 |
|------|------|------|
| `event_bus` | `EventBus` | 事件总线，负责事件的订阅与发布 |
| `services` | `ServiceManager` | 服务管理器，管理所有可插拔服务 |
| `api` | `BotAPI` | Bot API 接口（延迟初始化，服务加载后可用） |
| `dispatcher` | `EventDispatcher` | 事件分发器（延迟初始化） |
| `plugin_loader` | `PluginLoader` | 插件加载器（启动阶段创建） |

### 1.3 单例模式实现

```python
class BotClient:
    _initialized = False

    def __init__(self):
        if BotClient._initialized:
            return  # 已初始化则跳过
        BotClient._initialized = True
        # ... 初始化逻辑

    @classmethod
    def reset_singleton(cls):
        """重置单例状态（仅用于测试）"""
        cls._initialized = False
```

### 1.4 初始化流程

```
BotClient.__init__()
    │
    ├── 1. 创建 EventBus（默认超时 120 秒）
    │
    ├── 2. 创建 ServiceManager
    │       ├── 注册 MessageRouter
    │       ├── 注册 PreUploadService
    │       ├── 注册 PluginConfigService
    │       ├── 注册 RBACService
    │       ├── 注册 UnifiedRegistryService
    │       ├── 注册 FileWatcherService
    │       ├── 注册 PluginDataService
    │       └── 注册 TimeTaskService
    │
    └── 3. 初始化 LifecycleManager（传入 services, event_bus, registry）
```

### 1.5 公共接口

```python
# 获取已注册的插件列表
bot.get_registered_plugins() -> List[BasePlugin]

# 通过类型获取插件实例（类型安全）
bot.get_plugin(MyPlugin) -> MyPlugin

# 通过名称获取插件类
bot.get_plugin_class_by_name("my_plugin") -> Type[BasePlugin]
```

## 2. EventBus — 事件总线

EventBus 是框架的消息中枢，实现了发布-订阅模式，负责所有事件的路由和分发。

> 源码位置：`ncatbot/core/client/event_bus.py`

### 2.1 三种匹配模式

EventBus 支持三种灵活的事件匹配方式：

#### 精确匹配
事件类型完全相等时触发。

```python
# 只处理群消息事件
event_bus.subscribe("ncatbot.message.group", handler)
```

#### 前缀匹配
事件类型以订阅的前缀开头时触发。这意味着订阅 `"ncatbot.notice"` 可以捕获所有通知类事件。

```python
# 捕获所有以 "ncatbot.notice" 开头的事件
# 包括 ncatbot.notice.group_increase、ncatbot.notice.group_decrease 等
event_bus.subscribe("ncatbot.notice", handler)
```

#### 正则匹配
以 `"re:"` 前缀注册，使用正则表达式匹配事件类型。

```python
# 匹配所有 ncatbot 下的消息事件
event_bus.subscribe("re:ncatbot\\.message\\..*", handler)
```

### 2.2 订阅与发布

```python
# 订阅事件，返回 handler_id（UUID）
handler_id = event_bus.subscribe(
    event_type="ncatbot.message.group",
    handler=my_handler,        # Callable[[NcatBotEvent], Any]
    priority=10,               # 优先级，数值越大越先执行
    timeout=30.0,              # 超时时间（秒），None 使用默认值
    plugin=my_plugin,          # 可选，关联的插件实例
)

# 取消订阅
event_bus.unsubscribe(handler_id)

# 发布事件（异步）
results = await event_bus.publish(event)

# 线程安全发布（从非异步上下文）
future = event_bus.publish_threadsafe(event)

# 线程安全发布并等待结果
results = event_bus.publish_threadsafe_wait(event, timeout=5.0)
```

### 2.3 优先级与执行顺序

当多个处理器匹配同一事件时，按以下规则排序：

1. **优先级数值越大越先执行**
2. **同优先级按处理函数名字母序排序**
3. **任何处理器可调用 `event.stop_propagation()` 中止后续处理器执行**

```python
@bot.on_group_message(priority=100)  # 最先执行
async def high_priority(msg):
    ...

@bot.on_group_message(priority=0)    # 默认优先级，后执行
async def normal_priority(msg):
    ...
```

### 2.4 超时管理

每个处理器都有超时控制。超时时处理器会被中断，异常被记录但不影响其他处理器执行。

```python
event_bus.subscribe(
    "ncatbot.message.group",
    slow_handler,
    timeout=5.0,    # 5 秒超时
)
```

### 2.5 内部数据结构

```python
class EventBus:
    _exact: Dict[str, List[Tuple]]      # 精确匹配 {event_type: [(priority, handler, ...)]}
    _regex: List[Tuple]                  # 正则匹配 [(pattern, priority, handler, ...)]
    _handler_meta: Dict[UUID, Dict]      # 处理器元数据 {handler_id: {plugin, event_type, ...}}
    _loop: Optional[asyncio.AbstractEventLoop]  # 绑定的事件循环
```

## 3. NcatBotEvent — 事件封装

`NcatBotEvent` 是事件在 EventBus 中传递的统一封装。

> 源码位置：`ncatbot/core/client/ncatbot_event.py`

### 3.1 属性

| 属性 | 类型 | 说明 |
|------|------|------|
| `type` | `str` | 事件类型标识符（如 `"ncatbot.message.group"`） |
| `data` | `Any` | 事件数据对象（如 `GroupMessageEvent` 实例） |
| `results` | `List[Any]` | 各处理器的返回结果 |
| `exceptions` | `List[Exception]` | 处理过程中的异常 |
| `_propagation_stopped` | `bool` | 是否已停止传播 |

### 3.2 控制方法

```python
event = NcatBotEvent(type="ncatbot.message.group", data=msg_event)

# 停止事件传播（后续处理器不再执行）
event.stop_propagation()

# 拦截事件（等效于 stop_propagation）
event.intercept()

# 添加处理结果
event.add_result(result)

# 添加异常
event.add_exception(exception)
```

## 4. EventDispatcher — 事件分发器

`EventDispatcher` 负责将原始的 OneBot 数据转换为事件对象，并发布到 EventBus。

> 源码位置：`ncatbot/core/client/dispatcher.py`

### 4.1 处理流程

```
原始 WebSocket 数据 (dict)
    │
    ▼
EventDispatcher.dispatch(data)
    │
    ├── 1. 提取 post_type（message / notice / request / meta_event）
    │
    ├── 2. 提取二级类型（message_type / notice_type / request_type / meta_event_type）
    │
    ├── 3. 调用 EventParser.parse(data) 生成事件对象
    │       └── 根据 (post_type, secondary_key) 查找注册的事件类
    │           └── 使用 Pydantic 模型验证并创建事件对象
    │
    ├── 4. 为事件对象注入 API 实例（bind_api）
    │
    ├── 5. 构建事件类型字符串（如 "ncatbot.message.group"）
    │
    └── 6. 创建 NcatBotEvent 并发布到 EventBus
```

### 4.2 事件类型命名规范

事件类型字符串遵循 `ncatbot.<post_type>.<sub_type>` 的命名规范：

| 事件类型字符串 | 说明 |
|---------------|------|
| `ncatbot.message.group` | 群消息 |
| `ncatbot.message.private` | 私聊消息 |
| `ncatbot.notice.group_increase` | 群成员增加 |
| `ncatbot.notice.group_decrease` | 群成员减少 |
| `ncatbot.notice.group_ban` | 群禁言 |
| `ncatbot.notice.group_admin` | 群管理员变动 |
| `ncatbot.notice.group_upload` | 群文件上传 |
| `ncatbot.notice.group_recall` | 群消息撤回 |
| `ncatbot.notice.friend_add` | 好友添加 |
| `ncatbot.notice.friend_recall` | 好友消息撤回 |
| `ncatbot.notice.notify` | 系统通知（戳一戳等） |
| `ncatbot.request.friend` | 好友申请 |
| `ncatbot.request.group` | 加群请求 |
| `ncatbot.meta_event.lifecycle` | 生命周期事件 |
| `ncatbot.meta_event.heartbeat` | 心跳事件 |

## 5. EventRegistry — 事件注册接口

`EventRegistry` 为开发者提供了便捷的装饰器接口来注册事件处理器。

> 源码位置：`ncatbot/core/client/registry.py`

### 5.1 装饰器接口

```python
bot = BotClient()

# 群消息处理器
@bot.on_group_message(priority=0)
async def handle_group(msg: GroupMessageEvent):
    await msg.reply(text="收到群消息")

# 私聊消息处理器
@bot.on_private_message()
async def handle_private(msg: PrivateMessageEvent):
    await msg.reply(text="收到私聊消息")

# 通知事件处理器
@bot.on_notice()
async def handle_notice(event: NoticeEvent):
    ...

# 请求事件处理器
@bot.on_request()
async def handle_request(event: RequestEvent):
    ...

# 通用事件订阅
@bot.subscribe("ncatbot.notice.group_increase")
async def handle_join(event: NcatBotEvent):
    ...
```

### 5.2 自动参数解包

当处理器参数类型标注为具体的事件类（如 `GroupMessageEvent`），EventRegistry 会自动从 `NcatBotEvent.data` 中提取事件对象并传入：

```python
# 框架自动从 NcatBotEvent 中提取 GroupMessageEvent 对象
@bot.on_group_message()
async def handler(msg: GroupMessageEvent):
    # msg 已经是 GroupMessageEvent 实例，不是 NcatBotEvent
    print(msg.raw_message)
```

### 5.3 过滤器支持

```python
# 只处理特定群的消息
@bot.register_handler(
    event_type="ncatbot.message.group",
    handler=my_handler,
    filter_func=lambda event: event.data.group_id == "12345"
)
```

## 6. LifecycleManager — 生命周期管理

`LifecycleManager` 管理 Bot 从启动到关闭的完整生命周期。

> 源码位置：`ncatbot/core/client/lifecycle.py`

### 6.1 三种启动方式

```python
bot = BotClient()

# 方式一：同步阻塞启动（命令行场景）
bot.run(bot_uin="123456")

# 方式二：异步非阻塞启动（推荐，可在协程中使用）
api = await bot.run_backend_async(bot_uin="123456")
# api 可用于后续操作，Bot 在后台运行

# 方式三：同步环境下的非阻塞启动（独立线程）
api = bot.run_backend(bot_uin="123456", daemon=True)
# Bot 在守护线程中运行
```

### 6.2 启动参数

```python
class StartArgs(TypedDict, total=False):
    bot_uin: Union[str, int]           # QQ 号（必填）
    root: Optional[str]                # 管理员 QQ
    ws_uri: Optional[str]              # WebSocket 地址
    webui_uri: Optional[str]           # WebUI 地址
    ws_token: Optional[str]            # WebSocket Token
    webui_token: Optional[str]         # WebUI Token
    ws_listen_ip: Optional[str]        # WebSocket 监听 IP
    remote_mode: Optional[bool]        # 远程模式
    enable_webui: Optional[bool]       # 启用 WebUI
    debug: Optional[bool]              # 调试模式
    mock: Optional[bool]               # Mock 模式（测试用）
    load_plugin: Optional[bool]        # 是否加载插件
```

### 6.3 完整启动流程

```
bot.run(**kwargs)
    │
    ▼
_prepare_startup(**kwargs)                    [阶段一：准备]
    ├── 验证并更新启动参数到 ConfigManager
    ├── 检查配置问题（交互式修复）
    ├── 创建 PluginLoader
    └── NapCatService.launch()                 ← 启动/连接 NapCat
        ├── [本地模式] 安装 → 配置 → 启动 → 登录
        └── [远程模式] 检查远程服务状态
    │
    ▼
_core_execution()                             [阶段二：核心执行]
    ├── event_bus.bind_loop()                  ← 绑定事件循环
    ├── await services.load_all()              ← 加载所有注册的服务
    │       ├── MessageRouter (建立 WebSocket 连接)
    │       ├── PreUploadService
    │       ├── PluginConfigService
    │       ├── RBACService
    │       ├── UnifiedRegistryService
    │       ├── FileWatcherService
    │       ├── PluginDataService
    │       └── TimeTaskService
    ├── _setup_api()                           ← 初始化 BotAPI 和 Dispatcher
    ├── plugin_loader.load_external_plugins()  ← 加载用户插件
    ├── plugin_loader.load_builtin_plugins()   ← 加载内置插件
    ├── startup_event.set()                    ← 标记启动完成
    └── await router.websocket.listen()        ← 开始监听（阻塞）
    │
    ▼
_cleanup()                                    [阶段三：清理]
    ├── plugin_loader.unload_all()             ← 卸载所有插件
    └── services.close_all()                   ← 关闭所有服务
```

### 6.4 Mock 模式

Mock 模式用于测试，跳过 NapCat 服务的启动和 WebSocket 连接：

```python
bot.run(bot_uin="123456", mock=True)
```

在 Mock 模式下：
- 不启动 NapCat 服务
- 不建立 WebSocket 连接
- API 调用被模拟
- 适合单元测试和插件开发调试

### 6.5 安全关闭

```python
# 异步关闭
await bot.shutdown()

# 退出信号
bot.bot_exit()
```

关闭时会依次：
1. 卸载所有插件（调用 `on_close` 钩子）
2. 关闭所有服务
3. 清理 EventBus
4. 断开 WebSocket 连接
